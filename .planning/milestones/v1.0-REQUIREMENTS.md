# Requirements: Patent Citation Tool

**Defined:** 2026-02-27
**Core Value:** Highlight text on Google Patents, get an accurate citation reference instantly -- no PDF downloading, no manual counting.

## v1 Requirements

Requirements for initial release. Each maps to roadmap phases.

### Extension Foundation

- [x] **EXT-01**: Extension activates on Google Patents pages (patents.google.com)
- [x] **EXT-02**: Extension identifies patent/application number from the current page

### Citation -- Granted Patents

- [x] **CITE-01**: User can highlight text in the specification on Google Patents and receive a column:line citation (e.g., 4:5-20)
- [x] **CITE-02**: Citation correctly identifies the column number within the two-column patent specification
- [x] **CITE-03**: Citation correctly identifies the start and end line numbers within the column
- [x] **CITE-04**: When highlighted text spans a column boundary, a range citation is produced (e.g., 4:55-5:10)

### Citation -- Published Applications

- [x] **PAPP-01**: User can highlight text in a published application on Google Patents and receive a paragraph citation (e.g., [0045])
- [x] **PAPP-02**: When highlighted text spans multiple paragraphs, a range citation is produced (e.g., [0045]-[0047])

### PDF Parsing

- [x] **PDF-01**: Extension fetches the patent PDF linked from Google Patents
- [x] **PDF-02**: Extension extracts text with position data from the PDF's existing OCR/text layer (no client-side OCR)
- [x] **PDF-03**: Extension detects the two-column specification section, skipping cover page, preliminary material, and figures
- [x] **PDF-04**: Extension builds a column/line map by clustering text items by x-coordinate into columns

### Text Matching

- [x] **MATCH-01**: Extension performs fuzzy matching between Google Patents HTML text and PDF OCR text to locate the highlighted passage
- [x] **MATCH-02**: Extension displays a confidence indicator when match quality is uncertain (low-confidence warning)

### Output

- [x] **OUT-01**: Citation is copied to clipboard with a single click
- [x] **OUT-02**: A toast notification confirms the citation was copied

## v2 Requirements

Deferred to future release. Tracked but not in current roadmap.

### UX Enhancements

- **UX-01**: Configurable citation format (4:5-20 vs col. 4, ll. 5-20 vs column 4, lines 5-20)
- **UX-02**: Keyboard shortcut for citation (e.g., Ctrl+Shift+C)
- **UX-03**: Batch citation mode -- queue multiple citations and copy all at once
- **UX-04**: Configurable interaction mode -- popup-with-copy or auto-clipboard

### Caching

- **CACHE-01**: Local browser cache (IndexedDB) for previously analyzed patents
- **CACHE-02**: Cloud/shared cache so analyzed patents benefit all users
- **CACHE-03**: Patent family cache reuse -- continuation patents share specification text

### Data Sources

- **DATA-01**: USPTO eGrant API fallback when Google Patents PDF lacks OCR layer

## Out of Scope

| Feature | Reason |
|---------|--------|
| Client-side OCR | Massively increases extension size, slow, inaccurate on two-column layouts. Only ~2-5% of patents lack text layers. |
| Citation management/organization | Turns single-purpose tool into half-baked reference manager. Users paste into their own systems. |
| AI-powered patent summarization | Different product category entirely. Stay focused on one thing done well. |
| Non-US patent support | Completely different document formats and citation conventions. Multiplies complexity 3-5x. |
| Mobile/tablet support | Chrome extensions don't run on mobile. Attorneys draft on desktop. |
| Inline PDF viewer/annotator | Google Patents already shows the PDF. Major scope creep. |
| Full patent number in citation | Attorney already knows which patent they're citing. Optional in v2 as part of configurable format. |
