# Roadmap: Patent Citation Tool

## Overview

This roadmap delivers a Chrome extension that lets patent professionals highlight text on Google Patents and receive precise column:line or paragraph citations instantly. The critical path runs through PDF parsing -- the extension must fetch, parse, and map patent PDFs before any citation can be generated. The four phases follow this dependency chain: validate the MV3 extension architecture and PDF fetch, build the PDF parsing pipeline, generate citations via text matching, and deliver the complete clipboard output workflow.

## Phase Details

### Phase 1: Extension Foundation and PDF Fetch
**Goal**: A working MV3 extension skeleton that activates on Google Patents, identifies the patent number, and can fetch the patent PDF in the offscreen document -- confirming that CSP, CORS, and service worker lifecycle constraints are resolved
**Depends on**: Nothing (first phase)
**Requirements**: EXT-01, EXT-02, PDF-01
**Success Criteria** (what must be TRUE):
  1. Extension icon activates (lights up) only when the user is on a patents.google.com page
  2. Extension correctly extracts the patent or application number from the current Google Patents URL
  3. Extension successfully fetches a patent PDF from Google Patents in the offscreen document without CSP or CORS errors
  4. Service worker, content script, and offscreen document communicate via message passing without dropped messages

### Phase 2: PDF Parsing Pipeline
**Goal**: The extension can take any US granted patent PDF and produce an accurate PositionMap -- a data structure mapping every text span to its page, column, and line number within the two-column specification
**Depends on**: Phase 1
**Requirements**: PDF-02, PDF-03, PDF-04
**Success Criteria** (what must be TRUE):
  1. Extension extracts text items with x,y position data from a patent PDF's existing text layer using PDF.js
  2. Extension correctly identifies where the two-column specification section begins (skipping cover page, preliminary material, and figures)
  3. Extension clusters text items into left and right columns by x-coordinate and assigns accurate line numbers by y-coordinate sorting
  4. PositionMap is accurate when validated against a test set of 5+ patents from different decades and formats

### Phase 3: Text Matching and Citation Generation
**Goal**: When a user highlights text on Google Patents, the extension locates that text in the PositionMap and returns a correctly formatted citation -- column:line for granted patents, paragraph number for published applications
**Depends on**: Phase 2
**Requirements**: CITE-01, CITE-02, CITE-03, CITE-04, PAPP-01, PAPP-02, MATCH-01, MATCH-02
**Success Criteria** (what must be TRUE):
  1. User highlights text in a granted patent specification on Google Patents and receives a column:line citation (e.g., 4:5-20)
  2. When highlighted text spans a column boundary, a range citation is produced (e.g., 4:55-5:10)
  3. User highlights text in a published application on Google Patents and receives a paragraph citation (e.g., [0045]) or range (e.g., [0045]-[0047])
  4. Extension displays a low-confidence warning when the fuzzy match between HTML text and PDF text is uncertain
  5. Fuzzy matching handles common HTML-to-PDF divergences (smart quotes, dashes, ligatures, whitespace) without false mismatches

### Phase 4: Citation Output
**Goal**: The complete user workflow works end-to-end -- highlight text, get citation, have it on the clipboard with visual confirmation
**Depends on**: Phase 3
**Requirements**: OUT-01, OUT-02
**Success Criteria** (what must be TRUE):
  1. User can copy the generated citation to clipboard with a single click
  2. A toast notification confirms the citation was copied, visible near the user's selection
  3. The full workflow (highlight -> citation -> clipboard) completes without requiring the user to open a popup or navigate away from the page
