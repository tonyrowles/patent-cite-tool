**src/shared/constants.js** — Export shared constants:
```javascript
const MSG = {
  PATENT_PAGE_DETECTED: 'patent-page-detected',
  PDF_LINK_FOUND: 'pdf-link-found',
  PDF_LINK_NOT_FOUND: 'pdf-link-not-found',
  FETCH_PDF: 'fetch-pdf',
  PDF_FETCH_RESULT: 'pdf-fetch-result',
  GET_STATUS: 'get-status',
};

const STATUS = {
  IDLE: 'idle',
  FETCHING: 'fetching',
  READY: 'ready',
  ERROR: 'error',
  UNAVAILABLE: 'unavailable',
};

const PATENT_TYPE = {
  GRANT: 'grant',
  APPLICATION: 'application',
};
```
Use a pattern compatible with both service worker and content script (no ES modules — use simple object assignments on `globalThis` or a self-contained const block that can be loaded via importScripts or script tag, OR just duplicate the constants in each file since there are only 3 small objects). Choose the simplest approach: define constants directly in each file that uses them, referencing this file as the canonical source. Actually, the simplest MV3-compatible approach: make constants.js a plain JS file with top-level `const` declarations, and import it via the manifest's `content_scripts.js` array (listed before content-script.js) and via `importScripts('shared/constants.js')` in the service worker.

**Wait — `importScripts` is not available in MV3 service workers by default.** The correct MV3 approach for sharing code in service workers is to either: (a) use a single service worker file with constants inlined, or (b) use ES modules with `"type": "module"` in the manifest's background field. Use approach (b): set `background: { service_worker: "background/service-worker.js", type: "module" }` in manifest.json, and make constants.js an ES module with `export`. The content script cannot use ES modules, so for the content script, list constants.js before content-script.js in the `content_scripts.js` array — this makes the constants available as globals.

Update manifest accordingly:
- `background: { service_worker: "background/service-worker.js", type: "module" }`
- `content_scripts[0].js: ["shared/constants.js", "content/content-script.js"]`

For constants.js, write it so it works in both contexts:
- Define constants as `const MSG = { ... }` etc. (works as global in content script)
- Also `export { MSG, STATUS, PATENT_TYPE }` at the bottom (works as ES module import in service worker)

**Icons** — Generate simple placeholder PNG icons using a canvas-based approach or just create minimal valid PNG files. Use solid colored squares: blue for active (16x16, 48x48, 128x128), gray for inactive (16x16, 48x48, 128x128). The simplest approach: use a bash script with ImageMagick (`convert`) if available, or create minimal 1-color PNGs via a small Node.js script, or use Python. Check what's available and use the simplest tool. If nothing is available, create the smallest valid PNG files possible (a single-pixel PNG is ~67 bytes and can be written as base64-decoded binary).

<task type="auto">
  <name>Task 2: Implement service worker with declarativeContent rules and message handling</name>
  <files>src/background/service-worker.js</files>
  <action>
Create `src/background/service-worker.js` as an ES module.

**Import constants:**
```javascript
import { MSG, STATUS, PATENT_TYPE } from '../shared/constants.js';
```

**Declarative content rules** (in `chrome.runtime.onInstalled` listener):
1. Call `chrome.action.disable()` to disable icon by default
2. Remove all existing declarativeContent rules, then add one rule:
   - Condition: `PageStateMatcher` with `pageUrl: { hostEquals: 'patents.google.com', pathPrefix: '/patent/US', schemes: ['https'] }`
   - Action: `ShowAction()`

Per user decision: icon stays inactive on non-patent pages (search results, scholar pages, other patents.google.com URLs). The `pathPrefix: '/patent/US'` ensures only US patent pages activate.

**Message listener** (top-level `chrome.runtime.onMessage.addListener`):
Handle these message types:
- `MSG.PDF_LINK_FOUND`: Store patent info + pdfUrl in `chrome.storage.local` with status `FETCHING`. Log to console: `[SW] PDF link found for {patentId}`. (Actual fetch delegation to offscreen doc comes in Plan 02.)
- `MSG.PDF_LINK_NOT_FOUND`: Store patent info in `chrome.storage.local` with status `UNAVAILABLE`. Set badge: `chrome.action.setBadgeText({ text: '!' })` and `chrome.action.setBadgeBackgroundColor({ color: '#F59E0B' })` (amber). Per user decision: distinct "PDF unavailable" state.
- `MSG.GET_STATUS`: Read `currentPatent` from `chrome.storage.local` and call `sendResponse` with it. Return `true` to keep channel open.

Always return `true` from the onMessage listener to support async responses.

**State storage format** in `chrome.storage.local` key `currentPatent`:
```javascript
{
  patentId: string,      // e.g. "US11427642B2"
  patentType: string,    // "grant" or "application"
  kindCode: string,      // e.g. "B2", "A1"
  pdfUrl: string | null, // PDF download URL or null
  status: string,        // from STATUS constants
  error: string | null   // error message if status is ERROR
}
```

**Important anti-patterns to avoid:**
- Do NOT store state in global variables (service worker terminates after 30s idle). Always use chrome.storage.local.
- Register ALL message listeners at the top level of the file, not inside callbacks.

<task type="auto">
  <name>Task 3: Implement content script for patent extraction and PDF link discovery</name>
  <files>src/content/content-script.js</files>
  <action>
Create `src/content/content-script.js`. This file runs after `shared/constants.js` in the content script context, so `MSG`, `STATUS`, and `PATENT_TYPE` are available as globals.

**extractPatentInfo()** function:
- Parse `window.location.pathname` with regex `/\/patent\/(US[\dA-Z]+)/`
- If no match, return null
- Extract the kind code suffix with `/([A-Z]\d?)$/` from the patent ID
- Determine type: if kind code is A1, A2, or A9 → `PATENT_TYPE.APPLICATION`, else → `PATENT_TYPE.GRANT`
- Return `{ patentId, patentType, kindCode }`

Per user decisions:
- Extract patent number from URL path segment
- Ignore locale suffixes (/en), query parameters, and anchors (URL pathname parsing handles this)
- Detect and store patent type at extraction time

**findPdfLink()** function:
- Query `document.querySelectorAll('a[href*="patentimages.storage.googleapis.com"]')`
- Loop through results, return the first `link.href` that ends with `.pdf`
- Return null if none found

**waitForPdfLink(timeoutMs = 10000)** function:
- Check immediately with `findPdfLink()`
- If not found, set up a `MutationObserver` on `document.body` watching `{ childList: true, subtree: true }`
- On each mutation, check `findPdfLink()` — if found, disconnect observer and resolve
- Set a timeout at `timeoutMs` to disconnect and resolve with null
- Return a Promise

This handles the SPA pitfall where Google Patents renders the PDF link asynchronously.

**init()** async function:
1. Call `extractPatentInfo()` — if null, return (not a valid US patent page)
2. Await `waitForPdfLink()`
3. If pdfUrl found: send `{ type: MSG.PDF_LINK_FOUND, patentId, patentType, kindCode, pdfUrl }`
4. If pdfUrl null: send `{ type: MSG.PDF_LINK_NOT_FOUND, patentId, patentType, kindCode }`
5. Use `chrome.runtime.sendMessage()` for both

Per user decision: eager fetch — start as soon as patent page detected (don't wait for user interaction). The content script fires at `document_idle` and immediately begins looking for the PDF link.

**Call `init()` at the bottom of the file.**

**Important:** Do NOT use ES module syntax (import/export) in the content script. It runs in the page context where the manifest injects it as a classic script.
