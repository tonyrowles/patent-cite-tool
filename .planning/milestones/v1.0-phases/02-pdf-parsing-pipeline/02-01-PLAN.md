**Create src/offscreen/pdf-parser.js** -- PDF.js text extraction module:

```javascript
/**
 * PDF text extraction using PDF.js.
 *
 * Extracts text items with position data from each page of a patent PDF.
 * Returns per-page arrays of positioned text items for downstream
 * column/line analysis.
 */

import * as pdfjsLib from '../lib/pdf.mjs';

// Configure worker source for Chrome extension context
pdfjsLib.GlobalWorkerOptions.workerSrc = chrome.runtime.getURL('lib/pdf.worker.mjs');
```

**IMPORTANT:** The import syntax depends on what pdf.mjs actually exports. Check after vendoring in Task 1:
- If pdfjs-dist v5 uses named exports: `import { getDocument, GlobalWorkerOptions } from '../lib/pdf.mjs';`
- If it exports a default namespace: `import pdfjsLib from '../lib/pdf.mjs';`
- Adjust the code accordingly. The research suggests v5 uses named exports.

If the worker fails to load (e.g., CSP blocks it), PDF.js will fall back to main-thread parsing. This is acceptable for patent PDFs. Log a warning but do not fail.

**extractTextFromPdf(pdfData)** async function:
- `pdfData` is an ArrayBuffer
- Call `pdfjsLib.getDocument({ data: pdfData }).promise` (or `getDocument({ data: pdfData }).promise` with named import)
- First, check for text layer: call `hasTextLayer(pdf)` (see below). If false, throw `Error('NO_TEXT_LAYER')`.
- For each page (1 to pdf.numPages):
  - `const page = await pdf.getPage(i)`
  - `const textContent = await page.getTextContent()`
  - `const viewport = page.getViewport({ scale: 1.0 })`
  - Filter items: `textContent.items.filter(item => item.str && item.str.trim().length > 0)`
  - Map each item to: `{ text: item.str, x: item.transform[4], y: item.transform[5], width: item.width, height: item.height, fontName: item.fontName, hasEOL: item.hasEOL, pageNum: i, pageWidth: viewport.width, pageHeight: viewport.height }`
  - Collect into per-page result: `{ pageNum: i, items: pageItems, pageWidth: viewport.width, pageHeight: viewport.height }`
- Return array of per-page results
- Export the function

**hasTextLayer(pdf)** async function (internal):
- Check first `Math.min(pdf.numPages, 5)` pages
- Sum up text items (filtered for non-empty str) across those pages
- If total < 5, return false (no meaningful text layer)
- Return true

**Error handling:** Wrap the entire extraction in try/catch. Re-throw with descriptive messages. The `NO_TEXT_LAYER` error is a special case that the caller should handle to show the user-facing message "This patent PDF has no text layer. Citation not available."

chrome.runtime.onMessage.addListener((message, sender, sendResponse) => {

**parsePdf(patentId)** async function:
1. Open IndexedDB and read the PDF blob for the given patentId (reuse `openDb()` pattern from existing code)
2. Convert Blob to ArrayBuffer: `const arrayBuffer = await record.pdf.arrayBuffer()`
3. Call `extractTextFromPdf(arrayBuffer)`
4. Store the extracted text data back in IndexedDB alongside the PDF. Update the existing record: add a `textItems` field containing the per-page results array. Use `store.put()` to overwrite.
5. Send success message: `chrome.runtime.sendMessage({ type: PARSE_RESULT, success: true, patentId, pageCount: results.length })`
6. On error:
   - If error.message === 'NO_TEXT_LAYER': send `{ type: PARSE_RESULT, success: false, patentId, error: 'no-text-layer' }`
   - Otherwise: send `{ type: PARSE_RESULT, success: false, patentId, error: error.message }`

Keep all existing functions (fetchPdfWithRetry, openDb, storePdfInIndexedDB) intact. They remain the same, just now in ES module context (no behavior change).
