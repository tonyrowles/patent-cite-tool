<objective>
Create the Cloudflare Worker that acts as an API gateway between the Chrome extension and the USPTO Open Data Portal. The Worker holds the USPTO API key server-side, validates incoming requests with a bearer token, orchestrates the 3-step ODP lookup (search for application number, list documents to find EGRANT.PDF, download the PDF), and returns the PDF binary to the caller with proper CORS headers.

<task type="auto">
  <name>Task 2: Implement Worker fetch handler with auth, CORS, and ODP orchestration</name>
  <files>worker/src/index.js</files>
  <action>
Create `worker/src/index.js` as the single Worker entry point. The file exports a default object with one `fetch` handler.

**Request flow:**

1. **CORS preflight (OPTIONS):** Return 204 with headers:
   - `Access-Control-Allow-Origin: *`
   - `Access-Control-Allow-Methods: GET, OPTIONS`
   - `Access-Control-Allow-Headers: Authorization`
   - `Access-Control-Max-Age: 86400`

2. **Bearer token validation:** Check `Authorization` header matches `Bearer {env.PROXY_TOKEN}`. Return 401 if invalid. CRITICAL: Check OPTIONS *before* auth — preflight requests do not carry Authorization.

3. **Patent number extraction and validation:** Read `patent` query parameter. Strip `US` prefix and kind code suffix if present (per Pitfall 5 from RESEARCH.md — extension may send full ID like `US12505414B2`, Worker must extract digits). Validate remaining value matches `/^\d{6,8}$/`. Return 400 if invalid.

4. **ODP 3-step orchestration** (implement as `async function fetchEgrantPdf(patentNumber, apiKey)`):
   - **Step 1 — Search:** `GET https://api.uspto.gov/api/v1/patent/applications/search?q=applicationMetaData.patentNumber:{patentNumber}&offset=0&limit=25&fields=applicationNumberText` with `X-API-Key` header. Parse JSON. Extract `applicationNumberText` from `patentFileWrapperDataBag` or `results` array. Throw if empty.
   - **Step 2 — List documents:** `GET https://api.uspto.gov/api/v1/patent/applications/{appNumber}/documents?documentCodes=EGRANT.PDF` with `X-API-Key` header. Parse JSON. Find document matching `documentCode === 'EGRANT.PDF'` OR `documentDescription` containing `ELECTRONIC GRANT` or `ISSUE GRANT` (case-insensitive). Extract `downloadUrl` from `downloadOptionBag` (prefer `mimeTypeIdentifier === 'PDF'`, fallback to first entry). Throw if not found.
   - **Step 3 — Download:** `GET {downloadUrl}` with `X-API-Key` header. Return the Response object directly (stream body through to caller).

5. **Success response:** Return the PDF Response body with `Content-Type: application/pdf` and `Access-Control-Allow-Origin: *`.

6. **Error response:** Catch any error from the ODP flow. Return 502 with error message text and `Access-Control-Allow-Origin: *`. CRITICAL: CORS header on errors too (per Pitfall 2 from RESEARCH.md — without it, offscreen document gets opaque CORS error instead of HTTP status).

**Implementation notes:**
- Define a helper `corsHeaders()` that returns `{ 'Access-Control-Allow-Origin': '*' }` to ensure consistency on all responses.
- Use string template for ODP base URL: `const ODP_BASE = 'https://api.uspto.gov/api/v1/patent/applications'`
- The `fetchEgrantPdf` function takes `(patentNumber, apiKey)` and returns a Response whose body is the PDF stream.
- Patent number cleaning regex: `patentNumber.replace(/^US/i, '').replace(/[A-Z]\d*$/i, '')` — strips `US` prefix and kind code like `B1`, `B2`, `A1`.
