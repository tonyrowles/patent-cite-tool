**2. Implement `fetchUsptoWithRetry(patentId, retries = 1)`:**
This function mirrors the existing `fetchPdfWithRetry` pattern but calls the Worker instead of Google Patents.

```javascript
async function fetchUsptoWithRetry(patentId, retries = 1) {
  try {
    const workerUrl = `${WORKER_URL}?patent=${encodeURIComponent(patentId)}`;
    const response = await fetch(workerUrl, {
      headers: { 'Authorization': `Bearer ${PROXY_TOKEN}` }
    });
    if (!response.ok) {
      const errorText = await response.text().catch(() => `HTTP ${response.status}`);
      throw new Error(errorText);
    }
    const blob = await response.blob();
    await storePdfInIndexedDB(patentId, blob);
    chrome.runtime.sendMessage({
      type: USPTO_FETCH_RESULT,
      success: true,
      patentId,
    });
  } catch (error) {
    if (retries > 0) {
      await new Promise(r => setTimeout(r, 1000));
      return fetchUsptoWithRetry(patentId, retries - 1);
    }
    chrome.runtime.sendMessage({
      type: USPTO_FETCH_RESULT,
      success: false,
      patentId,
      error: error.message,
    });
  }
}
```

**Key points:**
- Uses the existing `storePdfInIndexedDB(patentId, blob)` function — this is the critical link that makes UPTO-03 work. The PDF blob is stored under the same `patentId` key, so when the service worker subsequently sends PARSE_PDF, the existing `parsePdf()` function reads it from IndexedDB and runs the identical extraction pipeline.
- One retry with 1s delay (same as Google fetch pattern).
- On error, reads response text for better error messages from the Worker (which returns descriptive error text like "USPTO lookup failed: Patent 12345678 not found in ODP").
- The offscreen document is subject to CORS (unlike the service worker), which is why the Worker must return `Access-Control-Allow-Origin: *` headers (handled in Plan 01).
- Do NOT change the `reasons` array in `ensureOffscreenDocument()` — the existing `reasons: ['BLOBS']` is sufficient for this fetch+blob operation (per Pitfall 6 from RESEARCH.md).

Place the function between `fetchPdfWithRetry` and the IndexedDB section (around line 199), keeping the file's existing organization of: message listener -> PDF parsing -> PDF fetch -> USPTO fetch -> IndexedDB -> citation lookup -> text matching.
