<task type="auto">
  <name>Task 1: Deploy Worker and set production secrets</name>
  <files>worker/</files>
  <action>
Deploy the Cloudflare Worker to production using the Wrangler CLI.

**Step 1: Install dependencies**
```bash
cd worker/ && npm install
```

**Step 2: Check wrangler auth**
```bash
npx wrangler whoami
```
If not authenticated, this will fail. The executor should run `npx wrangler login` which opens a browser for Cloudflare OAuth. If running in a headless environment, user will need to authenticate separately.

**Step 3: Deploy Worker**
```bash
npx wrangler deploy
```
This outputs the Worker URL (e.g., `https://patent-cite-worker.{subdomain}.workers.dev`). Capture this URL — it is needed for the next task.

**Step 4: Set production secrets**
The user must provide the actual secret values. Prompt the user for:
1. `PROXY_TOKEN` — a strong random string (e.g., `openssl rand -hex 32`) for extension-to-Worker auth
2. `USPTO_API_KEY` — their ODP API key from https://data.uspto.gov/

Set each secret:
```bash
echo "{value}" | npx wrangler secret put PROXY_TOKEN
echo "{value}" | npx wrangler secret put USPTO_API_KEY
```

**Step 5: Verify deployment**
```bash
curl -s -o /dev/null -w "%{http_code}" https://patent-cite-worker.{subdomain}.workers.dev
```
Should return 401 (no auth token = unauthorized). This confirms the Worker is live and validating tokens.

```bash
curl -s -o /dev/null -w "%{http_code}" -X OPTIONS https://patent-cite-worker.{subdomain}.workers.dev
```
Should return 204 (CORS preflight succeeds without auth).

  </verify>
  <done>Worker is deployed to *.workers.dev, PROXY_TOKEN and USPTO_API_KEY secrets are set, unauthenticated GET returns 401, OPTIONS returns 204 with CORS headers</done>
