**worker/src/index.js changes:**

2. **Update env type annotation** in the fetch handler JSDoc to include `PATENT_CACHE`:

3. **Extend the OPTIONS preflight handler** to allow POST method and Content-Type header:
   ```javascript
   'Access-Control-Allow-Methods': 'GET, POST, OPTIONS',
   'Access-Control-Allow-Headers': 'Authorization, Content-Type',
   ```

4. **Add cache route handling AFTER auth check, BEFORE existing patent number extraction.** The cache routes use the same `cleanPatentNumber()` and auth pattern but branch on pathname:

   ```javascript
   // After auth check (step 2), before existing patent extraction (step 3):
   const url = new URL(request.url);
   const path = url.pathname;

   // Cache routes
   if (path === '/cache') {
     // ... cache GET/POST handlers below
   }

   // Existing USPTO proxy route (path === '/' with patent query param)
   // ... existing code continues unchanged
   ```

   **Important:** Move the existing `const url = new URL(request.url);` line to before the cache routing check (it's currently at step 3). The rest of the existing code should remain unchanged.

5. **GET /cache handler** — Returns cached position map or 404:
   - Extract `patent` and `v` (default 'v1') from query params
   - Clean patent number with `cleanPatentNumber()`
   - Validate with `/^\d{6,8}$/` — return 400 on invalid (with CORS headers)
   - Construct key: `${version}:${patentNumber}` (e.g. `v1:12505414`)
   - Read from KV: `await env.PATENT_CACHE.get(key, { type: 'json' })`
   - Return 404 with CORS headers if null
   - Return 200 with JSON body and `Content-Type: application/json` + CORS headers if found

6. **POST /cache handler** (CACH-04: existence check before write):
   - Extract `patent` and `v` from query params (same as GET)
   - Clean and validate patent number (same as GET) — return 400 on invalid
   - Construct key: `${version}:${patentNumber}`
   - **Existence check:** `await env.PATENT_CACHE.get(key)` — if not null, return 200 'Already cached' with CORS headers (idempotent, not 409)
   - Parse request body: wrap `await request.json()` in try/catch — return 400 'Invalid JSON body' on parse failure (with CORS headers)
   - Write: `await env.PATENT_CACHE.put(key, JSON.stringify(payload))`
   - Return 201 'Cached' with CORS headers

   **Note on existence check:** Use `env.PATENT_CACHE.get(key)` (without `{ type: 'json' }`) for the existence check — we only need to know if it's null, not parse the value. This is slightly more efficient.

7. **Method guard for /cache path:** If request.method is neither GET nor POST on the /cache path, return 405 Method Not Allowed with CORS headers.

**Routing structure after changes:**
```
OPTIONS → CORS preflight (all routes)
Bearer token check → 401 on failure
/cache + GET → cache read
/cache + POST → cache write with existence check
/ + GET → existing USPTO proxy (unchanged)
anything else → 404 (or falls through to existing behavior)
```

**Anti-patterns to avoid:**
- Do NOT use `expirationTtl` on `put()` — no TTL per locked decision
- Do NOT use `type: 'json'` on the existence check read — just checking for null
- Do NOT forget CORS headers on ANY response branch (400, 404, 405, 200, 201)
