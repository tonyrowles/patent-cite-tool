<objective>
Wire the extension's service worker and offscreen document into the Cloudflare KV cache. Before fetching any PDF, the extension checks the cache for a pre-parsed position map. On a cache hit, the patent jumps directly to citation-ready state. On a miss, the existing PDF pipeline runs unchanged. After any successful local parse, the position map is uploaded to the cache (fire-and-forget).

Purpose: This completes the client-side integration so that one user's parse benefits all future users of that patent. The 3-second timeout ensures the cache layer is invisible when the Worker is unreachable.

Output: Updated service-worker.js with cache check and upload delegation, updated offscreen.js with cache read/write/IndexedDB operations.
</objective>

```javascript
function openDb() {
  return new Promise((resolve, reject) => {
    const request = indexedDB.open('patent-cite-tool', 1);
    request.onupgradeneeded = (event) => {
      const db = event.target.result;
      if (!db.objectStoreNames.contains('pdfs')) {
        db.createObjectStore('pdfs', { keyPath: 'patentId' });
      }
    };
    request.onsuccess = (event) => resolve(event.target.result);
    request.onerror = (event) => reject(new Error(`IndexedDB open failed: ${event.target.error}`));
  });
}
```

4. **Add checkCache() function** with 3-second AbortController timeout:
   ```javascript
   async function checkCache(patentId) {
     const controller = new AbortController();
     const timeoutId = setTimeout(() => controller.abort(), 3000);

     try {
       const url = `${WORKER_URL}/cache?patent=${encodeURIComponent(patentId)}&v=${CACHE_VERSION}`;
       const response = await fetch(url, {
         headers: { 'Authorization': `Bearer ${PROXY_TOKEN}` },
         signal: controller.signal,
       });
       clearTimeout(timeoutId);

       if (!response.ok) return null;  // 404 = cache miss, other errors = fallthrough
       return await response.json();
     } catch (err) {
       clearTimeout(timeoutId);
       // AbortError (timeout) or network error — fall through silently
       console.warn('[Offscreen] Cache check failed:', err.message);
       return null;
     }
   }
   ```

5. **Add handleCheckCache() function** — orchestrates cache check and routes to hit/miss:
   ```javascript
   async function handleCheckCache(patentId, pdfUrl) {
     const cached = await checkCache(patentId);

     if (cached && cached.entries && cached.meta) {
       await handleCacheHit(patentId, cached);
     } else {
       chrome.runtime.sendMessage({
         type: CACHE_MISS,
         patentId,
         pdfUrl,  // pass through for service worker to continue PDF fetch
       });
     }
   }
   ```

6. **Add handleCacheHit() function** — stores positionMap in IndexedDB, signals service worker:
   ```javascript
   async function handleCacheHit(patentId, cachedData) {
     try {
       const db = await openDb();
       await new Promise((resolve, reject) => {
         const tx = db.transaction('pdfs', 'readwrite');
         const store = tx.objectStore('pdfs');
         store.put({
           patentId,
           pdf: null,             // No PDF blob on cache hit
           timestamp: Date.now(),
           positionMap: cachedData.entries,
           positionMapMeta: cachedData.meta,
         });
         tx.oncomplete = () => { db.close(); resolve(); };
         tx.onerror = (e) => { db.close(); reject(e); };
       });

       chrome.runtime.sendMessage({
         type: CACHE_HIT_RESULT,
         patentId,
         lineCount: cachedData.meta.totalLines,
         columnCount: cachedData.meta.totalColumns,
       });
     } catch (err) {
       console.warn('[Offscreen] Cache hit IndexedDB write failed:', err.message);
       // Fall through to PDF fetch on IndexedDB failure
       chrome.runtime.sendMessage({
         type: CACHE_MISS,
         patentId,
         pdfUrl: null,  // pdfUrl not available here; service worker will need to handle
       });
     }
   }
   ```

   **Critical (Pitfall 4):** The positionMap MUST be written to IndexedDB, not just service worker state. The `lookupPosition()` function reads from IndexedDB exclusively. Without this write, cache hit would result in 'no-position-map' errors.

- Do NOT skip the `clearTimeout(timeoutId)` in the catch block — Pitfall 3 from research
- Do NOT remove the existing FETCH_PDF message sending — it's now triggered by handleCacheMiss instead of handlePdfLinkFound directly

2. **Add uploadToCache() function:**
   ```javascript
   async function uploadToCache(patentId) {
     try {
       const db = await openDb();
       const record = await new Promise((resolve, reject) => {
         const tx = db.transaction('pdfs', 'readonly');
         const store = tx.objectStore('pdfs');
         const request = store.get(patentId);
         request.onsuccess = () => resolve(request.result);
         request.onerror = (event) => reject(new Error(`IndexedDB read: ${event.target.error}`));
         tx.oncomplete = () => db.close();
       });

       if (!record?.positionMap || !record?.positionMapMeta) {
         console.warn('[Offscreen] No positionMap to upload for', patentId);
         return;
       }

       // Strip bounding box fields per locked decision — only cache:
       // text, column, lineNumber, page, section, hasWrapHyphen
       const entries = record.positionMap.map(({ text, column, lineNumber, page, section, hasWrapHyphen }) => ({
         text, column, lineNumber, page, section, hasWrapHyphen,
       }));

       const payload = {
         entries,
         meta: {
           totalLines: record.positionMapMeta.totalLines,
           totalColumns: record.positionMapMeta.totalColumns,
           hasClaimsSection: record.positionMapMeta.hasClaimsSection,
         },
         cachedAt: Date.now(),
         version: CACHE_VERSION,
       };

       const url = `${WORKER_URL}/cache?patent=${encodeURIComponent(patentId)}&v=${CACHE_VERSION}`;
       await fetch(url, {
         method: 'POST',
         headers: {
           'Authorization': `Bearer ${PROXY_TOKEN}`,
           'Content-Type': 'application/json',
         },
         body: JSON.stringify(payload),
       });
       // Response ignored — fire-and-forget per design
     } catch (err) {
       // Silently ignore — cache upload failure must never affect user
       console.warn('[Offscreen] Cache upload failed:', err.message);
     }
   }
   ```

   **Key details from locked decisions:**
   - Strip x, y, width, height from each entry — only keep text, column, lineNumber, page, section, hasWrapHyphen
   - Include positionMapMeta (totalLines, totalColumns, hasClaimsSection) as `meta`
   - Include `cachedAt` timestamp and `version` ('v1') metadata
   - Wrap entire function in try/catch — never throw (fire-and-forget)
   - Add console.warn in catch for development visibility (Pitfall 7 from research)

**Anti-patterns to avoid:**
- Do NOT await the upload in handleParseResult — it's fire-and-forget
- Do NOT include bounding box fields (x, y, width, height) in the cache payload — per locked decision
- Do NOT skip the console.warn in the catch block — needed for development debugging per Pitfall 7
