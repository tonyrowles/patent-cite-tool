  <files>worker/wrangler.toml</files>
  <action>
1. **Create the KV namespace** using wrangler CLI:
   ```bash
   cd worker/
   npx wrangler kv namespace create PATENT_CACHE
   ```
   This outputs the namespace ID. Copy the `id` value from the output.

2. **Replace the placeholder in wrangler.toml:**
   Update `id = "PLACEHOLDER_NAMESPACE_ID"` with the real namespace ID from step 1.

   **Pitfall 6 from research:** Forgetting to paste the namespace ID causes `wrangler deploy` to succeed but Worker throws `TypeError: Cannot read properties of undefined (reading 'get')` on KV access. Verify the ID is pasted before deploying.

3. **Deploy the Worker:**
   ```bash
   cd worker/
   npx wrangler deploy
   ```

4. **Verify deployment with a quick smoke test** — cache miss should return 404:
   ```bash
   curl -s -o /dev/null -w "%{http_code}" \
     -H "Authorization: Bearer 4509b9943f831fb140eb0c3a7304f23cc6f72e41b5e5f8c800a42e94f09cadbe" \
     "https://patent-cite-worker.fatduck.workers.dev/cache?patent=12505414&v=v1"
   ```
   Expected: 404 (no cache entry yet). If 200, the namespace was pre-populated. If 500, the KV binding is not correctly configured.

5. **Verify existing USPTO proxy route still works** (regression check):
   ```bash
   curl -s -o /dev/null -w "%{http_code}" \
     -H "Authorization: Bearer 4509b9943f831fb140eb0c3a7304f23cc6f72e41b5e5f8c800a42e94f09cadbe" \
     "https://patent-cite-worker.fatduck.workers.dev/?patent=12505414"
   ```
   Expected: 200 (existing USPTO proxy still returns PDF).

**Note:** If `wrangler deploy` or `wrangler kv namespace create` fails with an auth error, create a checkpoint for the user to authenticate. The deployment requires Cloudflare account credentials already configured from Phase 6.
