# Requirements: Patent Citation Tool

**Defined:** 2026-03-02
**Core Value:** Highlight text on Google Patents, get an accurate citation reference instantly — no PDF downloading, no manual counting.

## v1.2 Requirements

Requirements for v1.2 Store Polish + Accuracy Hardening. Each maps to roadmap phases.

### Testing

- [x] **TEST-01**: Automated test harness using Vitest that imports pure matching/parsing functions from existing modules
- [x] **TEST-02**: Fixture generation script that captures PositionMap JSON from real patent PDFs
- [x] **TEST-03**: Diverse fixture corpus covering 30-50 test cases (pre-2000, chemical, cross-column, repetitive claims, short/long selections, claims vs spec)
- [x] **TEST-04**: Frozen golden output baseline recorded before any algorithm changes
- [x] **TEST-05**: Off-by-one line detection in test output (distinguish systematic offset from total mismatch)
- [x] **TEST-06**: Documented accuracy metrics (citation match rate, exact accuracy, confidence calibration)

### Accuracy

- [x] **ACCY-01**: Manual accuracy audit across broad patent sampling identifies failure patterns
- [x] **ACCY-02**: Algorithm fixes for highest-impact failure modes found during audit
- [x] **ACCY-03**: All algorithm fixes validated against regression harness (no existing passing cases broken)

### Icons

- [x] **ICON-01**: Professional icon set at 16/32/48/128px for both active and inactive variants
- [x] **ICON-02**: Toolbar icon shows gray on non-patent pages, partial color when patent detected, full color when position map is parsed and ready
- [x] **ICON-03**: Icon generation script using sharp for reproducible builds from source SVG

### Options

- [x] **OPTS-01**: Options page accessible via right-click extension menu (`options_ui` manifest entry)
- [x] **OPTS-02**: Settings moved from popup to options page (trigger mode, display mode, patent prefix)
- [x] **OPTS-03**: "Saved" confirmation feedback after settings changes
- [x] **OPTS-04**: Privacy policy link and extension version displayed in options page

### Store

- [x] **STOR-01**: Privacy policy written and hosted at a stable public URL
- [ ] **STOR-02**: At least one 1280x800 screenshot showing citation in context on a real patent page
- [ ] **STOR-03**: 440x280 promotional tile for Chrome Web Store search
- [x] **STOR-04**: Store listing text — title (≤45 chars), summary (≤132 chars), description
- [x] **STOR-05**: Developer Dashboard privacy section complete (all four subsections)

## Future Requirements

Deferred to future release. Tracked but not in current roadmap.

### Citation Features

- **CITE-01**: Configurable citation format (4:5-20 vs col. 4, ll. 5-20 vs column 4, lines 5-20)
- **CITE-02**: Keyboard shortcut for citation (e.g., Ctrl+Shift+C)
- **CITE-03**: Batch citation mode — queue multiple citations and copy all at once
- **CITE-04**: Patent family cache reuse — continuation patents share specification text

## Out of Scope

Explicitly excluded. Documented to prevent scope creep.

| Feature | Reason |
|---------|--------|
| Full ESM module unification / build step | Simpler `export` keyword approach sufficient for v1.2; full build pipeline deferred |
| International patent support (EP, JP, PCT) | Multiplies complexity 3-5x; different formats and conventions |
| AI-powered summarization | Different product category |
| Mobile browser support | Chrome desktop extension only |
| Batch citation mode | Not essential for core prosecution workflow |
