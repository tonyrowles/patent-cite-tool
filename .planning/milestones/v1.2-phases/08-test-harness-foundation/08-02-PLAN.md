<task type="auto">
  <name>Task 1: Create fixture generation script</name>
  <files>scripts/generate-fixture.js</files>
  <action>
Create `scripts/generate-fixture.js` — a Node.js script that:

1. Accepts a patent ID as CLI argument: `node scripts/generate-fixture.js US5959167`
2. Also accepts an optional `--batch` flag to process a list from stdin or a file.

3. Stubs `globalThis.chrome` BEFORE any dynamic imports:
   ```javascript
   globalThis.chrome = {
     runtime: { getURL: (path) => new URL(path, import.meta.url).href }
   };
   ```
   This is required because `pdf-parser.js` calls `chrome.runtime.getURL('lib/pdf.worker.mjs')` at module scope.

4. Dynamically imports `extractTextFromPdf` from `../src/offscreen/pdf-parser.js` and `buildPositionMap` from `../src/offscreen/position-map-builder.js`.

5. Constructs the PDF URL: `https://patentimages.storage.googleapis.com/pdfs/${patentId}.pdf`
   - If this URL 404s, try the alternate URL format: `https://patentimages.storage.googleapis.com/${patentId}.pdf`
   - If both fail, report the error clearly and exit with code 1.

6. Fetches the PDF using native `fetch()` (Node 18+ built-in).

7. Passes the ArrayBuffer to `extractTextFromPdf`, then the result to `buildPositionMap`.

8. Validates the resulting PositionMap:
   - If fewer than 50 entries, print a WARNING: "PositionMap has only N entries — likely a scanned PDF without text layer. Skipping."
   - If 0 entries, skip the file entirely.

9. Writes the PositionMap JSON to `tests/fixtures/${patentId}.json` with 2-space indentation.

10. Prints summary: "Wrote N entries to tests/fixtures/US5959167.json"

IMPORTANT: If pdf-parser.js cannot be imported in Node.js due to PDF.js browser-specific code, use `pdfjs-dist` npm package as a fallback:
   - `npm install --save-dev pdfjs-dist`
   - Import `getDocument` from `pdfjs-dist/legacy/build/pdf.mjs`
   - Re-implement the extraction logic directly in the script (extracting text items with position data from each page)
   - Still use `buildPositionMap` from position-map-builder.js for the second stage

The fixture generation script is a dev tool — it runs once to populate fixtures, not during test execution. Network access is acceptable here.
