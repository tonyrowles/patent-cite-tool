Off-by-one classification tiers (from CONTEXT.md decisions):
- **exact**: citation matches golden exactly
- **systematic**: start and end both off by same ±1 amount (line-counting bug pattern)
- **boundary**: only start or end off by ±1 (selection boundary ambiguity pattern)
- **mismatch**: column mismatch or ±2 or more in any direction
- **no-match**: algorithm returned null when golden expected a citation (or vice versa)

  <action>
1. Create `tests/helpers/classify-result.js`:

   Export a `classifyResult(expected, actual)` function that:
   - Parses citation strings in formats:
     - Same-column: `"col:start-end"` (e.g., `"4:15-20"`)
     - Cross-column: `"startCol:startLine-endCol:endLine"` (e.g., `"3:45-4:5"`)
     - Single-line: `"col:line"` (e.g., `"4:15"`)
   - Returns `{ tier: string, detail: string | null }` where tier is one of:
     - `'exact'` — citation matches exactly (delta_start=0, delta_end=0)
     - `'systematic'` — start and end both off by the same ±1 (suggests line-counting bug)
     - `'boundary'` — only start or end off by ±1, not both by same amount (suggests selection boundary ambiguity)
     - `'mismatch'` — column mismatch OR any line delta ≥ ±2
   - Handle nulls: if either expected or actual is null/undefined, return `{ tier: 'mismatch', detail: 'null citation' }`
   - Handle unparseable citations: return `{ tier: 'mismatch', detail: 'unparseable' }`

   Classification logic per CONTEXT.md decisions:
   - Columns must match exactly (column off = always mismatch)
   - Compute `delta_start = actual.startLine - expected.startLine`
   - Compute `delta_end = actual.endLine - expected.endLine`
   - If `delta_start === 0 && delta_end === 0` → `exact`
   - If `abs(delta_start) <= 1 && abs(delta_end) <= 1`:
     - If `delta_start === delta_end && delta_start !== 0` → `systematic` (shifted by same amount)
     - Else → `boundary` (one side off or both off by different ±1)
   - Else → `mismatch`

2. Create a unit test file `tests/unit/classify-result.test.js` for the classifier with the behavior cases listed above.
