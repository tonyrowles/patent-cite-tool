**Purpose:** The locked decision (CONTEXT.md Audit Methodology) mandates: "Follow with a quick live spot-check on 10-15 real patents via Google Patents to catch HTML/PDF divergences fixtures can't simulate." This task implements that requirement.

**Step 1: Prepare the extension for testing**

Load the extension in Chrome (`chrome://extensions` → Load unpacked → select the project root). Ensure the extension is active and functional.

**Step 2: Select 10-15 real patents for spot-check**

Open 10-15 patent pages on Google Patents (`patents.google.com/patent/USXXXXXXX`). Select patents that cover a range of types:
- 3-4 modern patents (2010+)
- 2-3 pre-2000 patents
- 1-2 chemical/biotech patents with formulas or special characters
- 1-2 patents with long dependent claim chains
- 1-2 continuation or reissue patents
- Mix of patents already in the corpus AND patents NOT in the corpus

**Step 3: Perform the spot-check on each patent**

For each patent:
1. Open the patent page in Chrome with the extension active
2. Select text of varying length from the patent page:
   - Short selection (1-2 lines from specification body)
   - Medium selection (a full paragraph)
   - Claims selection (one full claim)
3. Trigger the citation tool and record:
   - Patent number
   - Selected text (exact HTML copy)
   - Citation output returned by the extension
   - Tier classification: correct / off-by-one / wrong / no-match
4. Note any HTML/PDF divergences observed (e.g., different whitespace, character encoding, reflow artifacts)

**Step 4: Capture failures as test cases**

For EACH failure (wrong citation or no-match) discovered during the spot-check:
1. Generate a fixture if one doesn't already exist:
   ```bash
   node scripts/generate-fixture.js USXXXXXXX
   ```
2. Add a new test case entry to `tests/test-cases.js` with:
   - `id`: `USXXXXXXX-spotcheck-{category}` (e.g., `US9876543-spotcheck-claims`)
   - `patentFile`: path to the generated fixture
   - `selectedText`: the EXACT text copied from Google Patents HTML during the spot-check
   - `category`: appropriate category from CATEGORIES
3. Add the failure as an audit report entry with: patent number, selected text, expected vs actual, failure category, proposed root cause

**Step 5: Update golden baseline**

After adding all new test cases from spot-check failures:
```bash
npm run update-golden
npx vitest run
```

Confirm no regressions on previously-passing cases. New failure cases will show as their current (broken) state in the golden — these become targets for Plan 09-02.

**Step 6: Record spot-check results**

Document the full spot-check results (all 10-15 patents, not just failures) in the plan summary. Include:
- Total patents checked
- Total selections tested
- Pass rate (correct citations / total selections)
- List of HTML/PDF divergences discovered
- Number of new test cases added from failures
