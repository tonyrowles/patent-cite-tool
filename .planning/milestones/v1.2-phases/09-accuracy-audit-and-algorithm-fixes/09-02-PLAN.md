<!-- filterGutterLineNumbers current implementation -->
From src/offscreen/position-map-builder.js:
```javascript
export function filterGutterLineNumbers(items, boundary, pageWidth) {
  const pageMid = pageWidth / 2;
  return items.filter(item => {
    const text = item.text.trim();
    if (!/^\d{1,2}$/.test(text)) return true;
    const num = parseInt(text, 10);
    if (num < 5 || num > 65 || num % 5 !== 0) return true;
    const nearBoundary = Math.abs(item.x - boundary) <= 40;
    const nearCenter = Math.abs(item.x - pageMid) <= 40;
    if (!nearBoundary && !nearCenter) return true;
    return false;
  });
}
```

In `src/content/text-matcher.js`, in or near `normalizeText()` or as a preprocessing step in `matchAndCite()`:

```javascript
// Strip HTML-copy wrap hyphens: "insti- tutions" -> "institutions"
// A hyphen followed by a space then a lowercase letter is a wrap artifact
// in HTML-copied text, not a real hyphen (compare "well-known" = no space)
.replace(/- ([a-z])/g, '$1')
```

IMPORTANT caveats:
- This transform should only apply to the selected text (user's HTML copy), not to the concat (PDF-derived)
- Apply AFTER whitespace collapse so `"insti-  tutions"` becomes `"insti- tutions"` first
- Only matches `"- "` followed by lowercase (not `"-k"` which would be a real hyphen like in `"well-known"`)
