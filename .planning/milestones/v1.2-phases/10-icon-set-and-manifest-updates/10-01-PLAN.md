**Step 2: Create the source SVG at `src/icons/icon-source.svg`**

Design a professional icon with these requirements:

- **Motif:** A document/page with column lines and a bracket or highlight indicator — conveying "citation from a patent document." Think: a simplified patent page with a highlighted line or bracket annotation. The motif should read as "text reference tool" at a glance.
- **ViewBox:** Use `viewBox="0 0 256 256"` (large enough so sharp downscales to all sizes with crisp results — see Pitfall 4 in research).
- **CSS classes for state coloring:** Use named CSS classes on SVG path/rect elements so the generation script can inject different color schemes per state:
  - `.icon-primary` — the main fill color (document body, page)
  - `.icon-accent` — the accent/highlight color (bracket, highlight bar, citation indicator)
  - `.icon-detail` — fine detail lines (column lines, text lines)
- **Design for 16px legibility:** At 16x16, the icon must be recognizable. Avoid fine details that disappear at small sizes. Use bold shapes, clear silhouette, high contrast. Test mentally: would this be recognizable as a tiny favicon?
- **No text in SVG:** Text in SVGs renders inconsistently across systems and at small sizes.

The SVG should use simple geometric shapes (rect, path, line) — not complex bezier art. Patent-themed: a page silhouette with a column divider line and a highlighted/bracketed line region suggesting "this text is cited."

**Step 3: Create the generation script at `scripts/generate-icons.mjs`**

Create an ESM script that:

1. Reads `src/icons/icon-source.svg` as a UTF-8 string
2. Defines three color schemes corresponding to the three icon states:
   - **Active (full color):** Vibrant blue primary (`#2563eb` or similar), warm accent for the highlight/bracket (`#f59e0b` amber or `#10b981` emerald), dark detail lines. This is the "ready to cite" state — should look alive and inviting.
   - **Inactive (gray):** All elements in grayscale. Primary `#9ca3af`, accent `#d1d5db`, detail `#e5e7eb`. Should read as "disabled/not available." Pure gray, no color tint.
   - **Partial (desaturated/muted):** Primary in a muted/washed-out version of the active color (`#93c5fd` light blue or similar), accent in a subtle muted tone (`#fcd34d` pale amber), detail in light gray. Should read as "working on it, not ready yet" — recognizably the same icon as active but clearly not fully saturated.
3. For each state, performs CSS injection by replacing the `.icon-primary`, `.icon-accent`, `.icon-detail` fill values in the SVG string using string replacement (NOT relying on sharp's `svg.stylesheet` option — see research Open Question 1 about librsvg version dependency).
4. For each state x size combination (3 x 4 = 12), runs:
   ```javascript
   await sharp(Buffer.from(modifiedSvg))
     .resize(size, size)
     .png()
     .toFile(outputPath);
   ```
5. Outputs the filename for each generated PNG to stdout.
6. Uses the naming convention: `icon-{active|inactive|partial}-{16|32|48|128}.png`

Color injection approach: Replace CSS class fill values via string replace on the SVG source. For example:
```javascript
function applyColorScheme(svgString, scheme) {
  return svgString
    .replace(/\.icon-primary\s*\{[^}]*\}/g, `.icon-primary { fill: ${scheme.primary}; }`)
    .replace(/\.icon-accent\s*\{[^}]*\}/g, `.icon-accent { fill: ${scheme.accent}; }`)
    .replace(/\.icon-detail\s*\{[^}]*\}/g, `.icon-detail { fill: ${scheme.detail}; }`);
}
```

The SVG should define these classes in a `<style>` block with placeholder colors that get replaced.
