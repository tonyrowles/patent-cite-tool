<objective>
Refactor content scripts from globals pattern to ES module imports, create Firefox manifest, and prepare codebase for esbuild bundling.

Purpose: Content scripts currently rely on load-order globals (via constants-globals.js and text-matcher.js wrappers). esbuild requires explicit import/export statements to resolve the module graph. This plan converts all content scripts to ES modules, deletes the wrapper files, creates the Firefox manifest, and updates supporting config.

Output: Bundle-ready content scripts with explicit imports/exports, standalone Firefox manifest, updated Vitest config, dist/ in .gitignore.
</objective>

**Step 3: Create src/manifest.firefox.json**

Create a standalone Firefox MV3 manifest at `src/manifest.firefox.json`. This is NOT generated from the Chrome manifest -- it is a separate, complete manifest file per user decision.

Key differences from Chrome manifest:
- `browser_specific_settings.gecko.id` with a valid extension ID (e.g., `"patent-cite-tool@example.com"`)
- No `"offscreen"` permission (Firefox doesn't support offscreen documents)
- No `"declarativeContent"` permission (Firefox uses tabs.onUpdated instead)
- Background uses `"scripts": ["background/service-worker.js"]` instead of `"service_worker"` (Firefox MV3 syntax)
- Background includes `"type": "module"` (Firefox MV3 supports ESM background scripts)
- Content scripts array points to the bundled file: `["content/content.js"]` (same as dist/chrome will use)
- Keep same `host_permissions`, `action`, `options_ui`, `icons`, `web_accessible_resources` sections
- Permissions: `["activeTab", "storage", "contextMenus", "clipboardWrite"]` (drop declarativeContent, offscreen)

The manifest must be complete and accurate so Phase 16 does not need to modify it.
