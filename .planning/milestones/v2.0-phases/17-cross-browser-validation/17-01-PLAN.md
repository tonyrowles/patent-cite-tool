2. Extend `scripts/build.js` to build test-export bundles. Add a new function `buildTestExports()` after the existing build functions:
   ```javascript
   async function buildTestExports() {
     await Promise.all([
       esbuild.build({
         entryPoints: ['src/matching-exports.js'],
         bundle: true,
         format: 'esm',
         platform: 'browser',
         outfile: 'dist/chrome/matching-exports.js',
       }),
       esbuild.build({
         entryPoints: ['src/matching-exports.js'],
         bundle: true,
         format: 'esm',
         platform: 'browser',
         outfile: 'dist/firefox/matching-exports.js',
       }),
     ]);
   }
   ```
   Call `buildTestExports()` at the end of the `main()` function, AFTER both `buildChrome()` and `buildFirefox()` complete (or after whichever target is being built — if `--chrome-only`, build only `dist/chrome/matching-exports.js`; if `--firefox-only`, only `dist/firefox/matching-exports.js`). Do NOT call it in watch mode.

   The test exports are per-target because esbuild resolves differently per format/platform configuration. The Chrome bundle comes from the same source as the Chrome IIFE content bundle; the Firefox bundle from the same source as the Firefox ESM background bundle. This proves each target's bundling did not corrupt the matching logic.
