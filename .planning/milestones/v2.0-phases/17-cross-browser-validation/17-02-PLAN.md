  <files>scripts/spot-check.js</files>
  <action>
Create `scripts/spot-check.js` following the pattern from `scripts/accuracy-report.js`. The script:

1. Imports `matchAndCite` from `../src/shared/matching.js` and `TEST_CASES` from `../tests/test-cases.js` (dynamic imports since these are ESM).
2. Defines `SPOT_CHECK_IDS` array with the 5 representative patent test case IDs:
   - `US11427642-spec-short-1` (modern-short)
   - `US5440748-spec-long` (pre2000-long)
   - `US9688736-chemical-seq` (chemical)
   - `US11427642-cross-col` (cross-column)
   - `US7346586-claims-repetitive` (repetitive/claims)
3. For each spot-check ID:
   - Finds the test case in TEST_CASES by ID
   - Reads the fixture JSON file (positionMap)
   - Calls `matchAndCite(tc.selectedText, positionMap)`
   - Prints:
     - Patent number and Google Patents URL: `https://patents.google.com/patent/{PATENT_NUMBER}/en`
     - Category name
     - First 80 chars of selectedText (so user knows what to select)
     - Expected citation string from matchAndCite result
4. Prints a header explaining the verification process: load each URL in BOTH Chrome and Firefox, select the indicated text, confirm the extension citation matches the expected output.
5. At the end, prints a summary checklist:
   ```
   === VERIFICATION CHECKLIST ===
   For each patent above:
   [ ] Chrome: citation matches expected
   [ ] Firefox: citation matches expected
   [ ] Chrome and Firefox produce identical output
   ```

Run with: `node scripts/spot-check.js`
