## Decisions Made

1. **build:chrome (not build) in the e2e:smoke chain.** Smoke only consumes `dist/chrome/`; running the full Chrome+Firefox build is wasted work. Phase 26 smoke needs ~17 ms of build time; the full `build` would add ~600 ms of Firefox bundling for zero value. Phase 27 keeps this pattern.

2. **No explicit spec path argument.** The plan's locked script value is `playwright test --config tests/e2e/playwright.config.js` (no trailing `specs/smoke.spec.js`). Playwright auto-discovers under the config's `testDir: './specs'`. This means Phase 27 can add `replay.spec.js` etc. without script churn.

3. **Strict tri-condition shim probe.** The plan's `<action>` content for Task 1 (and the corresponding RESEARCH.md sketch) included `sr !== null && el.shadowRoot !== null && el.shadowRoot === sr`. I retained the third clause: it guarantees the override returned the SAME root (not just any open root, which would mask a broken override that creates a second root). The probe stays a single `page.evaluate` call.

4. **No deviation from the plan's exact spec text.** The PLAN provided the full file content verbatim. I matched it character-for-character (54 lines including comments, leading `//` header, named import order, `test.describe` wrapper). Conformance proved by the `<automated>` grep checks all passing.
