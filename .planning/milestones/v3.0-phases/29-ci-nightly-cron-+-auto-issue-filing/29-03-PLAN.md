Create `.github/workflows/e2e-nightly.yml` with this exact structure. Key decisions (already locked in CONTEXT.md):
- **Triggers:** `schedule: '0 6 * * *'` + `workflow_dispatch` with optional `force_full_suite` boolean input
- **Permissions:** `contents: read`, `issues: write` only
- **Concurrency group:** static string `e2e-nightly` (mitigates T-29-02-4 race — per 29-RESEARCH.md Risk 1, the CONTEXT.md's dynamic group expression is overridden here to a static name)
- **Timeout:** 30 minutes
- **Cache key:** `pw-${{ runner.os }}-${{ hashFiles('package.json') }}`
- **No `--with-deps`** on `playwright install chromium`
- **Pre-flight smoke** with `continue-on-error: true` then conditional meta-drift step
- **Regression step** with `continue-on-error: true` so the workflow can file issues then exit success
- **Artifact upload** with `if: failure() || steps.regression.outcome == 'failure' || steps.smoke.outcome == 'failure'` (the issue IS the signal, but we still want the artifact bundle uploaded for any failure)
- **Label creation** as an idempotent step before any `gh issue create` runs (29-RESEARCH.md Risk 6)

```yaml
# .github/workflows/e2e-nightly.yml
#
# Phase 29 (CRON-01, CRON-03) — Nightly E2E cron + auto-issue filing.
#
# Schedule: 06:00 UTC daily. Manual trigger via workflow_dispatch.
# Sunday runs the full live suite (66 cases); Mon-Sat run a deterministic
# 30-case rotating sample (see scripts/select-cron-cases.mjs).
# Smoke pre-flight on the seed patent gates the suite — on DOM drift, a
# single meta-issue is filed and the regression is skipped (no 76-issue storm).
# Per-case failures (non-FLAKE) are filed via scripts/e2e-report-issue.mjs
# with fingerprint-based dedup.
#
# The workflow ALWAYS exits success after issue filing — the filed issue is
# the failure signal, not a red X on the workflow run page.

name: E2E Nightly

on:
  schedule:
    - cron: '0 6 * * *'
  workflow_dispatch:
    inputs:
      force_full_suite:
        description: 'Run all 66 live cases (override day-of-week rotation)'
        type: boolean
        default: false

# Static concurrency group prevents schedule + workflow_dispatch from racing
# (per 29-RESEARCH.md Risk 1 — overrides CONTEXT.md's dynamic expression).
concurrency:
  group: e2e-nightly
  cancel-in-progress: false

permissions:
  contents: read
  issues: write

jobs:
  e2e-nightly:
    runs-on: ubuntu-latest
    timeout-minutes: 30
    env:
      PLAYWRIGHT_RUN_ID: ${{ github.run_id }}
      GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
      GITHUB_REPOSITORY: ${{ github.repository }}

    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: 22
          cache: ${{ hashFiles('package-lock.json') != '' && 'npm' || '' }}

      - name: Install dependencies
        run: |
          if [ -f package-lock.json ]; then
            npm ci
          else
            npm install
          fi

      - name: Cache Playwright Chromium
        id: playwright-cache
        uses: actions/cache@v4
        with:
          path: ~/.cache/ms-playwright
          key: pw-${{ runner.os }}-${{ hashFiles('package.json') }}

      - name: Install Playwright Chromium
        if: steps.playwright-cache.outputs.cache-hit != 'true'
        run: npx playwright install chromium

      - name: Verify Playwright binary
        run: npx playwright --version

      - name: Build Chrome extension
        run: npm run build:chrome

      - name: Run Phase 29 unit tests (sanity gate)
        run: npx vitest run tests/unit/select-cron-cases.test.js tests/unit/e2e-report-issue.test.js

      - name: Ensure e2e-nightly label exists
        run: gh label create "e2e-nightly" --color "0075ca" --description "Auto-filed by nightly E2E cron" --force 2>/dev/null || true

      - name: Pre-flight smoke (1 case)
        id: smoke
        continue-on-error: true
        run: |
          npx playwright test \
            --config tests/e2e/playwright.config.js \
            --grep "US11427642-spec-short-1" \
            specs/regression.spec.js

      - name: File meta-issue on smoke failure
        if: steps.smoke.outcome == 'failure'
        run: node scripts/e2e-report-issue.mjs --meta-drift

      - name: Select test cases for today
        id: cases
        if: steps.smoke.outcome == 'success'
        run: |
          if [ "${{ inputs.force_full_suite }}" = "true" ]; then
            CASES=$(node scripts/select-cron-cases.mjs --full)
          else
            CASES=$(node scripts/select-cron-cases.mjs)
          fi
          # Mask the pipe-OR'd ID list as a single output line; --grep accepts it as regex
          echo "cases=$CASES" >> "$GITHUB_OUTPUT"
          echo "Selected $(echo "$CASES" | tr '|' '\n' | wc -l) cases for this run"

      - name: Run E2E regression
        id: regression
        if: steps.smoke.outcome == 'success'
        continue-on-error: true
        run: |
          npx playwright test \
            --config tests/e2e/playwright.config.js \
            --grep "${{ steps.cases.outputs.cases }}" \
            specs/regression.spec.js

      - name: File issues for failures
        if: steps.smoke.outcome == 'success' && steps.regression.outcome == 'failure'
        run: node scripts/e2e-report-issue.mjs

      - name: Upload E2E artifacts
        if: always() && (steps.smoke.outcome == 'failure' || steps.regression.outcome == 'failure')
        uses: actions/upload-artifact@v4
        with:
          name: e2e-nightly-${{ github.run_id }}
          path: tests/e2e/artifacts/
          retention-days: 14
          if-no-files-found: warn
```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Actions runner → workflow YAML steps | Workflow steps execute with `GITHUB_TOKEN` scope and have network access; the YAML's permissions block constrains blast radius |
| Cron event → concurrent workflow_dispatch | Two trigger types can fire in the same window; concurrency group must encompass both |
| Workflow expressions (`${{ }}`) → shell | `${{ inputs.force_full_suite }}` etc. are interpolated into shell commands; only typed/restricted values flow through |
