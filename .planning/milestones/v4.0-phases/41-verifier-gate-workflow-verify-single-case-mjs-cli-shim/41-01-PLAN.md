<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Actions runner shell -> Node CLI | The diff-guard helper reads `git diff --name-only` output via stdin from the workflow; untrusted-source paths from a PR-author-controlled diff are validated against the regex bank. |
| GitHub PR API -> Node CLI | The affected-cases parser reads PR body content via stdin (workflow pipes `gh pr view --json body --jq '.body'`); the PR body is fully PR-author-controlled per Pitfall 1. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-41-01-T1 | Tampering | check-diff-guard.mjs FORBIDDEN_PATHS regex bank | mitigate | All 6 paths LOCKED per PITFALLS Pitfall 3 Defense 2; Vitest cases F1-F7 pin each; F12 prevents over-broad glob (e.g., v40-* matching non-v40 workflows); CODEOWNERS pins `scripts/` so unauthorized regex changes require @TR review (Phase 39 setup) |
| T-41-01-T2 | Tampering | parse-affected-cases.mjs regex (PR-body input) | mitigate | Regex is anchored `<!--\s*affected_cases\s*:\s*(...)\s*-->` and uses non-greedy `[\s\S]*?` — cannot be tricked into capturing arbitrary HTML by crafted PR bodies; split on safe delimiters `[,\n]`; trim + filter empties before returning. PR body NEVER reaches `run:` step as raw text (workflow uses stdin/env-hop pattern per Pitfall 1 step 4 + Phase 40-03 CR-02 lesson). |
| T-41-01-I1 | Information Disclosure | Helpers handle no secrets | accept | Both helpers are pure-function path/string processors. Zero access to env vars, fs, network, secrets. ESLint purity guard (Phase 42 owns; not applicable here — helpers don't yet have the restriction but their content is verified pure by Vitest). |
| T-41-01-E1 | Elevation of Privilege | CLI guard pattern executes user-supplied input | mitigate | Helpers process strings only; no `eval`, no `child_process`, no `fs.writeFile`. Exit codes are the ONLY side effect besides stdout/stderr writes. |

ASVS Categories addressed: V4 (Access Control — diff-guard enforces path-level access on the auto-fix agent), V5 (Input Validation — both helpers validate input shape with defensive parsing), V7 (Error Handling — exit codes 0/1 are meaningful, never silent).
</threat_model>
