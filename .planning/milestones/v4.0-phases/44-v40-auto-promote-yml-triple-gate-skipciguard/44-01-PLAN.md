## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub webhook → workflow runner | `pull_request.closed` event payload is attacker-controllable (PR author chooses body + title + branch name). Workflow must NEVER `${{ }}` interpolate raw payload into shell. |
| Auto-fix PR body → workflow parse step | `<!-- source_issue: N -->` comment is parsed for an integer that gates `gh issue view`. A crafted body could try to exfiltrate or DoS via injection. |
| Source-issue labels → triple-gate leg 3 | `gh issue view --json labels` returns structured JSON; safe as long as parsed as JSON (not regex-grepped). |
| auto-fix-promote.mjs → runPromote | Calls `_skipCiGuard:true`. This is the ONLY caller permitted to do so. Phase 35 designed the param for exactly this use case after triple-gate. |
| Auto-promote PR → tests/test-cases.js | A PR that opens with this branch prefix BYPASSES verifier-gate (Phase 41 only triggers on `auto-fix/*`). The CODEOWNER review is the human gate. |
| Workflow → main branch | NEVER direct-pushes. All mutations go through the auto-promote PR. |
