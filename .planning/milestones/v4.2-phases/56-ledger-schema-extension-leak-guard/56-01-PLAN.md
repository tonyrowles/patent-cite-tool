<objective>
Wire the `errorClass` ledger field into every `auto-fix.mjs` ledger write site (LEDGER-01), introduce the `safeAppendLedger` wrapper that enforces the CI/override leak guard at call-site scope (LEDGER-02), and add the integration test that catches partial errorClass wiring (LEDGER-04). This is the load-bearing change of Phase 56.

Purpose: The 7 direct `appendLedgerEntry(LEDGER_PATH, ...)` calls in `scripts/auto-fix.mjs` currently (a) bypass the Phase 48 PRE-02 guard (creating a leak vector where a local `--force-api` run can pollute the committed ledger) and (b) lack the `errorClass` field that `a-b-winner.mjs` requires to exit abstention. Both gaps must close in a single coherent change because the wrapper's introduction and the call-site rewrites have an inter-commit invariant: `safeAppendLedger` must exist before any call site references it, and once any call site references it, every direct `appendLedgerEntry(LEDGER_PATH, ...)` must be gone (otherwise the grep gate `grep -c 'safeAppendLedger({' = 7` is wrong mid-stream). RESEARCH §8 explicitly warns against splitting these.

Output: `scripts/auto-fix.mjs` with `safeAppendLedger` defined as a module-internal helper + 7 call-site rewrites; `tests/unit/auto-fix.test.js` with one new LEDGER-04 describe-block. Single atomic commit recommended: `feat(56): wire errorClass + safeAppendLedger leak guard (LEDGER-01, LEDGER-02, LEDGER-04)`.
