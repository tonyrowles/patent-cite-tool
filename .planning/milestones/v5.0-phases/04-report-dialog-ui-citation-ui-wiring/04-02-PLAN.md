<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user/host page → dialog DOM | note text + page-derived field values flow into the Shadow DOM panel and the outbound payload |
| dialog → background | the assembled payload crosses into the background via sendMessage; must carry no forbidden fields |
| Google Patents page → dialog keyboard/focus | host-page Polymer key bindings compete for Escape/Tab |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-06 | Tampering / XSS | dialog DOM construction | mitigate | All user-note and page-derived field values are rendered with `textContent` / createElement, never innerHTML, into the closed Shadow DOM (no markup injection). |
| T-04-07 | Information Disclosure | payload context assembly | mitigate | context is built only from the allowlisted PAY-09 fields + low-fidelity browser/OS; ip/clientIp/userAgent are never set; buildReportPayload (Phase 2) drops unknown fields and the [Remove selection text] toggle omits selectionText (D-06). |
| T-04-08 | Elevation / focus hijack | focus trap + Escape | mitigate | shadowRoot keydown handler calls stopPropagation()+preventDefault() on Escape and stopPropagation() on Tab so Google Patents key bindings cannot steal the interaction (RESEARCH Pitfall 2); focus is trapped within the panel and restored on close (Pitfall 5). |
| T-04-09 | Spoofing | cross-origin POST from content script | mitigate | report-dialog.js routes exclusively through MSG.SUBMIT_REPORT to the background (XPORT-06); the closed shadow root never exposes the PROXY_TOKEN (token lives only in the background). Verified: no `fetch(WORKER_REPORT_URL` in the module. |
| T-04-SC | Tampering | npm/pip/cargo installs | mitigate | Zero new packages (DoD zero-dependency; jsdom intentionally NOT added). No install tasks. |
</threat_model>
