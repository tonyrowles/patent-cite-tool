<objective>
Refactor showReportDialog so the SAME function renders the report dialog in two mount contexts — the existing closed Shadow DOM (in-citation, Phase 4) and a plain page-DOM container (the options-page #report section, CAP-06) — without duplicating any form/payload/submit logic (D-08). Output: mountContext-threaded showReportDialog + installFocusTrapPage + page-mode mount/dismiss/click-outside/inline-confirmation paths; both citation-ui.js shadow callers updated.

It already guards extractPatentInfo with `typeof extractPatentInfo === 'function'`; this plan must preserve and extend that guard so the options bundle loads cleanly.
