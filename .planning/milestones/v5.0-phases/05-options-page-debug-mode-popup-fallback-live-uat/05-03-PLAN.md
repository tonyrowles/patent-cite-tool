## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| popup → chrome.storage.local (pendingOptionsHash) | Transient signal value written by popup, read+deleted by options page; trusted extension origin |
| chrome.storage.local currentPatent → options page DOM | The user's own last-citation snapshot rendered into the page (no Shadow DOM); patentId rendered via banner (.textContent in 05-02) |
| options dialog → background (MSG.SUBMIT_REPORT) | Payload crosses to background then Worker; unchanged from Phase 3/4 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-08 | Tampering | pendingOptionsHash storage value | accept | Value is read-then-deleted and only used to scroll to a section by element id; no dynamic code path; negligible (RESEARCH Security Domain) |
| T-05-09 | Information Disclosure | popup-path report leaks more than intended | mitigate | Context is sourced ONLY from currentPatent (the user's own last citation); selectionText is null on this path so the [Remove selection text] toggle is inert/hidden — nothing beyond the snapshot is transmitted |
| T-05-10 | Tampering | currentPatent.patentId rendered in options page DOM | mitigate | Banner rendered via .textContent (enforced in 05-02); options.js passes patentNumber as data only, no innerHTML |
| T-05-11 | Information Disclosure | Discord webhook URL in shipped extension | mitigate | DoD grep guard (verified in 05-05 UAT-04); this plan adds no Worker URL or webhook reference to extension code |
| T-05-SC | Tampering | npm/pip/cargo installs | mitigate | Zero new dependencies; no install tasks; nothing to slopcheck |
</threat_model>
