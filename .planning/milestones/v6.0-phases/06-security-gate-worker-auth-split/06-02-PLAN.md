<objective>
Restructure `worker/src/index.js` from a single global Bearer gate into per-route authentication so webapp callers authenticate by Origin header while extension callers keep Bearer — with zero observable change to extension behavior. Add a new Origin-authenticated `GET /webapp/pdf` PDF-proxy route, dual-auth on `GET /cache`, webapp provenance tagging on `POST /cache`, a per-IP webapp rate limit, a global daily KV-write guard, and a zero-I/O published-application rejection that runs first on every patent route.

Purpose: SEC-03/04/05 + WRKR-01/02/03/04 — open webapp-accessible Worker routes safely before any public webapp URL exists. This is the blocking security gate.
Output: A single restructured `worker/src/index.js` with new helpers and routes; all new KV-write paths honor `X-PCT-Test-Mode: true`.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| public internet → Worker (webapp routes) | Origin-header is the only auth signal; untrusted clients can forge it |
| webapp browser → POST /cache | webapp-supplied position-map JSON is written to shared KV |
| concurrent isolates → wq:YYYYMMDD counter | non-atomic read-modify-write at the 900/1000 quota margin |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-04 | Spoofing | Non-browser client forging `Origin: https://cite.tonyrowles.com` (curl) | accept | Origin auth is explicitly not a boundary vs non-browser clients (RESEARCH Pitfall 7); backstopped by checkWebappRateLimit (30/60s) and checkDailyWriteGuard (900/day). Residual risk accepted for v1 (pre-launch). |
| T-06-05 | Spoofing | CORS bypass via mismatched preflight vs actual-response origin | mitigate | OPTIONS reflects the matched origin (Task 2); webappCorsHeaders always pairs ACAO with Vary:Origin on every response incl. 4xx/5xx (Pitfall 1/5) |
| T-06-06 | Elevation of Privilege | Rate-limit evasion by IP rotation | mitigate | checkWebappRateLimit keys on `CF-Connecting-IP` (Cloudflare egress IP, not a spoofable client header); sufficient at free-tier scale |
| T-06-07 | Tampering | Webapp cache-poisoning via POST /cache uploads | mitigate | source:"webapp" provenance tag injected server-side (Task 2); existence check blocks overwrite of extension entries; daily write guard caps blast radius |
| T-06-08 | Denial of Service | KV write-quota exhaustion via POST /cache flood | mitigate | checkDailyWriteGuard hard-caps at 900/day; per-IP 30/60s limit slows the attack |
| T-06-09 | Tampering | Published-application number → plausible-but-wrong citation | mitigate | isPublishedApplication zero-I/O guard runs FIRST on raw input on every patent route → HTTP 400 before any USPTO fetch (Pitfall 2) |
| T-06-10 | Denial of Service | Daily-write-guard read-modify-write race at the 900/1000 margin | accept | KV has no atomic increment; the 100-write buffer (900 vs 1000) absorbs the bounded concurrent overshoot; acceptable by design (RESEARCH non-atomicity analysis) |
| T-06-SC | Tampering | npm installs | accept | Zero new packages this phase; no install tasks |
</threat_model>
