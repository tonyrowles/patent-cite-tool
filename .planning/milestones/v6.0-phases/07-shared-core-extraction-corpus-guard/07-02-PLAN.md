<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| page.route() synthetic origin → disk | The test serves repo files to a headless browser at a synthetic origin. Only files under the repo root are served; the route handler resolves requests to in-repo paths and aborts unknown paths (no traversal outside the repo, no network). |
| harness → configurePdfWorker('/src/lib/pdf.worker.mjs') | The harness injects a constant, same-origin worker path — modeling exactly the trusted-constant injection the Phase 8 webapp must use (the T-07-01 boundary from Plan 01). The test does NOT inject a user- or network-supplied URL. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-04 | Information Disclosure | page.route() disk-serving handler | mitigate | Handler maps request paths to a fixed allowlist of in-repo locations (src/shared, src/lib, the harness, the PDF fixture) and route.abort()s anything else; no path-traversal-derived absolute paths are served; the origin is synthetic and local-only (headless, no external network). |
| T-07-05 | Tampering | CORE-04 worker-thread assertion masked by main-thread fallback | mitigate | The harness calls configurePdfWorker BEFORE extractTextFromPdf and the spec asserts BOTH a valid citation (1:37) AND a spawned pdf.worker.mjs worker (Pitfall 3) — a silent main-thread fallback fails the worker assertion, so the test cannot pass without real off-main-thread execution. |
| T-07-SC | Tampering | npm/pip/cargo installs | accept | No package installs in this phase (Playwright already at 1.60.0). Package Legitimacy Gate N/A per RESEARCH.md. No checkpoint required. |
</threat_model>
