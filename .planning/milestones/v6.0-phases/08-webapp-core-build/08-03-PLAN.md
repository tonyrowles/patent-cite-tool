From webapp/js/normalizer.js (08-02):
  normalizePatentInput(raw) → normalized id (e.g. 'US10123456B2')
  isPublishedApplication(raw) → boolean
  formatCitationLong(startEntry, endEntry) → 'Col. X, ll. Y-Z' | 'Col. X, l. Y' | 'Col. X, l. Y – Col. X2, l. Z'
  applyPrefix(citation, normalizedPatentId, includePrefix) → prefixed-or-bare citation

From src/shared (Phase 7):
  configurePdfWorker(url)   — call ONCE at init with '/lib/pdf.worker.mjs'
  extractTextFromPdf(pdfData) → pageResults  (runs PDF.js in worker thread)
  buildPositionMap(pageResults) → positionMap array
  formatCitation(startEntry, endEntry) → shorthand '4:15-22'  (the SHORT path; long path uses normalizer)
  matchAndCite(selectedText, positionMap, '', '') → { citation, startEntry, endEntry, confidence } | null

Worker API (Phase 6 — RESEARCH Pattern 3 + Code Examples):
  WORKER_URL = 'https://pct.tonyrowles.com';  CACHE_VERSION = 'v5'
  GET  `${WORKER_URL}/cache?patent={enc}&v=v5`     → 200 { entries:[{text,column,lineNumber,page,section,hasWrapHyphen}], meta:{...} } | 404 miss
  GET  `${WORKER_URL}/webapp/pdf?patent={enc}`      → arrayBuffer (PDF bytes)
  POST `${WORKER_URL}/cache?patent={enc}&v=v5`      body { entries, meta, cachedAt, version:'v5' }  — fire-and-forget
  NONE of these carry an Authorization header. The browser sends Origin automatically. (SEC-03)

UI-SPEC State Machine (10 states) + Copywriting Contract — loading strings are LOCKED:
  'Fetching patent PDF…' | 'Parsing PDF…' | 'Matching passage…' | 'Loading from cache…'
Confidence chip thresholds (locked): ≥0.95 chip-green "High confidence"; ≥0.80 chip-yellow "Medium confidence"; <0.80 chip-red "Low confidence".

localStorage keys (UI-SPEC component 10): 'citation-format' ('short'|'long', default 'short'); 'include-patent-number' ('true'|'false', default 'false').

DOM ids present in index.html (08-01): patent input, field-error <p>, passages section, "add another passage" button, submit (#cite-btn), #results-area, #citation-format, #include-patent-number.

Cache-hit caveat (RESEARCH Pitfall 4): cached entries lack bounding-box x/y/width/height. The webapp only reads text/column/lineNumber/page/section/hasWrapHyphen — fine. Never read startEntry.x.
Re-render caveat (Pitfall 5): store { startEntry, endEntry, confidence, passage } per result in memory so toggle changes re-render without re-fetch.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| webapp client → Worker API | the only sanctioned network egress; must use Origin auth, never a Bearer token |
| webapp client → USPTO/patentimages | FORBIDDEN direct path (CORS + bypasses Worker rate limits/controls) |
| user input → clipboard/localStorage | only user-entered passages + derived citations stored; no secrets |

| T-08-06 | Spoofing/Tampering | PDF acquisition path | mitigate | Grep guard: zero `patentimages`/`googleapis`; all PDF fetches via Worker `/webapp/pdf` (APP-03) |

</threat_model>
