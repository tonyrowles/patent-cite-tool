<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| caller → configurePdfWorker(url) → GlobalWorkerOptions.workerSrc | The worker-source URL crosses from the app/entry layer into PDF.js. In this phase the ONLY callers are extension contexts where the URL comes from `chrome.runtime.getURL` (trusted, packaged asset). In Phase 8 the webapp injects its own static asset path — noted here so Phase 8 must inject a same-origin/constant path, never a user- or network-supplied URL. |
| relocation → matchAndCite / buildPositionMap algorithm | Behavior-preservation boundary: a silent algorithm change during the move would poison produced/cached position maps and corrupt citations. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Tampering | configurePdfWorker(url) → GlobalWorkerOptions.workerSrc | mitigate (boundary note for Phase 8) | This phase's only callers pass `chrome.runtime.getURL(...)` (packaged, trusted). configurePdfWorker performs no validation by design — it is a thin seam. Documented constraint carried to Phase 8: the webapp MUST inject a constant same-origin asset path, never an attacker-influenced URL, or it could load arbitrary worker code. No untrusted input reaches this seam in Phase 7. |
| T-07-02 | Tampering | byte-verbatim relocation of position-map-builder.js / pdf-parser.js | mitigate | Use `git mv` (not retype) so the algorithm is provably unchanged; CORE-03 corpus (76 cases, byte-identical baseline.json) + full `npm test` gate the move. Any accidental edit fails the corpus assertion. |
| T-07-03 | Denial of Service | pdf-parser.js import with no chrome global | mitigate | Removing the module-scope `chrome.runtime.getURL` call eliminates the import-time ReferenceError that previously made the module unusable (self-DoS) in non-extension contexts. import-safety smoke test enforces this. |
| T-07-SC | Tampering | npm/pip/cargo installs | accept | No package installs in this phase (zero new dependencies, locked decision). Package Legitimacy Gate N/A — RESEARCH.md confirms no installs. No checkpoint required. |
