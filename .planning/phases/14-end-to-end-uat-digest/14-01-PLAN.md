## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-01 | Tampering/Injection | renderBugReportsSection rendering Issue/PR titles+bodies into Markdown | mitigate | The section emits ONLY derived COUNT/ratio metrics — it does NOT echo untrusted Issue/PR titles or bodies into the digest. Body access is limited to the `source_issue` numeric regex (`\d+` only). Acceptance criterion forbids rendering raw title/body text. Any future row that surfaces text must escape Markdown control chars. |
| T-14-02 | Denial of Service | fetchBugReportIssues on gh failure | mitigate | Errors RETURNED not thrown (D-05); runDigest degrades the section to `n/a` and still ships the digest — a hostile/empty gh response cannot abort the Monday digest. Tested by the errors-returned degrade test. |
| T-14-03 | Information Disclosure | new secret surface in the digest cron | mitigate | GitHub-only source (D-01); negative YAML guard asserts no `wrangler`/`CLOUDFLARE` token enters the cron path — no CF creds added. |
| T-14-04 | Tampering | label-positional counting drift | mitigate | Label-membership counting via `names.includes(...)` (never `labels[0]`) — GH does not guarantee label array order; CR-01 discipline enforced in acceptance criteria. |
| T-14-SC | Tampering | npm/pip/cargo installs | accept | No new package installs in this plan — modifies existing files only; no dependency additions. |
</threat_model>
