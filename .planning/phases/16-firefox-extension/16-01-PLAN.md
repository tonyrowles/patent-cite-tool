  <name>Task 1: Create Firefox pdf-pipeline.js with IndexedDB degradation</name>
  <files>src/firefox/pdf-pipeline.js</files>
  <action>
Create `src/firefox/pdf-pipeline.js` that absorbs ALL logic from `src/offscreen/offscreen.js` but uses direct function calls instead of Chrome's message-passing pattern. This is the core of the Firefox port.

**Structure:**
1. Imports: `extractTextFromPdf` from `../offscreen/pdf-parser.js`, `buildPositionMap` from `../offscreen/position-map-builder.js`, `matchAndCite` from `../shared/matching.js`, `MSG` from `../shared/constants.js`
2. Constants: `WORKER_URL`, `PROXY_TOKEN`, `CACHE_VERSION` (same values as offscreen.js)
3. Module-level state: `idbAvailable` boolean (default true), `positionMapCache` Map for in-memory fallback

**IndexedDB degradation (FOX-05):**
- Keep `openDb()` function from offscreen.js
- On first real IndexedDB write failure (InvalidStateError), set `idbAvailable = false`
- When `idbAvailable` is false: skip all IndexedDB reads/writes, use `positionMapCache` Map (patentId -> { positionMap, positionMapMeta }) as in-memory fallback
- Silent degradation -- no user-facing indication. `console.warn` only.
- Core guarantee: citations work without IndexedDB, just no local caching

**Exported functions (called directly by background.js):**

`checkServerCache(patentId, pdfUrl)` -- Check Cloudflare KV cache. Returns `{ hit: true, lineCount, columnCount }` or `{ hit: false, pdfUrl }`. On cache hit, store positionMap in IndexedDB (or in-memory if idb unavailable). Absorbs `handleCheckCache()` + `handleCacheHit()` + `checkCache()` from offscreen.js.

`fetchAndParsePdf(patentId, pdfUrl)` -- Fetch PDF from Google URL with retry, store blob in IndexedDB (or skip if idb unavailable), parse with extractTextFromPdf, build positionMap, store positionMap in IndexedDB (or positionMapCache). Returns `{ success: true, lineCount, columnCount }` or `{ success: false, error }`. Absorbs `fetchPdfWithRetry()` + `parsePdf()` + `storePdfInIndexedDB()`.

`fetchUsptoAndParse(patentId)` -- Fetch from Cloudflare Worker proxy with retry, same flow as fetchAndParsePdf. Absorbs `fetchUsptoWithRetry()`.

`lookupPosition(selectedText, patentId, contextBefore, contextAfter)` -- Read positionMap from IndexedDB or positionMapCache, run matchAndCite, return result. Absorbs `lookupPosition()` from offscreen.js.

`uploadToCache(patentId)` -- Read positionMap from IndexedDB or positionMapCache, strip bounding box fields, POST to Cloudflare KV. Fire-and-forget. Absorbs `uploadToCache()` from offscreen.js.

**Key differences from offscreen.js:**
- No `chrome.runtime.onMessage.addListener` -- these are just exported async functions
- No `chrome.runtime.sendMessage` calls -- return values directly
- IndexedDB operations wrapped with idbAvailable check: when false, use positionMapCache Map
- `parsePdf` logic inlined into `fetchAndParsePdf` (no separate message round-trip needed)
- The `storePdfInIndexedDB` step stores the PDF blob AND later the positionMap in sequence (not separate message-triggered operations)

**CRITICAL:** The function signatures return results directly (Promises). The caller (background.js) handles status updates and icon changes -- pdf-pipeline.js is pure data pipeline.

**CRITICAL:** For `lookupPosition`, when IndexedDB is unavailable, read from `positionMapCache` Map instead. The Map key is `patentId`, value is the full record object `{ positionMap, positionMapMeta }`.

**Part A: Create `src/firefox/background.js`**

Thin orchestrator that mirrors Chrome's `src/background/service-worker.js` but replaces all offscreen document message-passing with direct function calls to pdf-pipeline.js.

**Imports:**
```javascript
import { MSG, STATUS, PATENT_TYPE } from '../shared/constants.js';
import { checkServerCache, fetchAndParsePdf, fetchUsptoAndParse, lookupPosition, uploadToCache } from './pdf-pipeline.js';
```

**Icon paths:** Same ICON_PATHS object as Chrome's service-worker.js (partial + active states). Same `setTabIcon(tabId, state)` helper.

**onInstalled listener (FOX-03):**
- `chrome.action.disable()` -- disable icon globally by default
- `chrome.contextMenus.create({ id: 'get-patent-citation', title: 'Get Citation', contexts: ['selection'], documentUrlPatterns: ['https://patents.google.com/patent/US*'] })`
- NO declarativeContent -- Firefox does not support it

**tabs.onUpdated listener (FOX-03):**
- `chrome.tabs.onUpdated.addListener((tabId, changeInfo) => { ... })`
- Guard: `if (!changeInfo.url) return;`
- If `changeInfo.url.startsWith('https://patents.google.com/patent/US')` -> `chrome.action.enable(tabId)`
- Else -> `chrome.action.disable(tabId)`

**contextMenus.onClicked listener:**
- Same pattern as Chrome's service-worker.js
- Send `MSG.GENERATE_CITATION` to content script tab

**onMessage listener (message routing):**
Handle same message types as Chrome, but call pipeline functions directly instead of forwarding to offscreen:

- `MSG.PDF_LINK_FOUND` -> `handlePdfLinkFound(message, tabId)`: Set storage to FETCHING + partial icon, call `checkServerCache()`. If hit: set PARSED + active icon. If miss and pdfUrl: call `fetchAndParsePdf()`. If miss and no pdfUrl: call `fetchUsptoAndParse()`. On success: set PARSED + active icon + fire-and-forget `uploadToCache()`. On failure: try USPTO fallback (if grant + not already tried), then set ERROR.
- `MSG.PDF_LINK_NOT_FOUND` -> `handlePdfUnavailable(message, tabId)`: If grant patent, check cache then try USPTO. If application, set UNAVAILABLE + amber badge.
- `MSG.LOOKUP_POSITION` -> `handleLookupPosition(message, sender)`: Call `lookupPosition()` directly, send CITATION_RESULT back to content tab.
- `MSG.GET_STATUS` -> `handleGetStatus(sendResponse)`: Read from storage, return true for async.

**Key differences from Chrome's service-worker.js:**
- No `ensureOffscreenDocument()` -- not needed
- No `MSG.PDF_FETCH_RESULT`, `MSG.PARSE_RESULT`, `MSG.USPTO_FETCH_RESULT`, `MSG.CACHE_HIT_RESULT`, `MSG.CACHE_MISS`, `MSG.UPLOAD_TO_CACHE` handlers -- these were internal offscreen<->service-worker messages. Firefox calls functions directly and gets return values.
- The `handlePdfLinkFound` function becomes an async flow: check cache -> (miss) fetch PDF -> parse -> update storage -> set icon -> upload cache. All in one function with try/catch, not split across multiple message handlers.
- Same `handlePdfUnavailable` logic: for grants, check cache first, then try USPTO if miss. For applications, set UNAVAILABLE immediately.
- Badge text/color logic identical to Chrome (red for error, amber for unavailable/no-text-layer)

**CRITICAL:** All event listeners (`onInstalled`, `tabs.onUpdated`, `onMessage`, `contextMenus.onClicked`) MUST be registered at the top level of the module -- not inside callbacks. Firefox MV3 event pages require top-level listener registration.

**Part B: Update `src/manifest.firefox.json`**

Add `"tabs"` to the permissions array (required for `tabs.onUpdated` `changeInfo.url` access).

Add `content_security_policy` top-level key:
```json
"content_security_policy": {
  "extension_pages": "script-src 'self' 'wasm-unsafe-eval'"
}
```

This is required for PDF.js WebAssembly execution in the Firefox background script context.
