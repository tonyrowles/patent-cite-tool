<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 2: Human verification of Firefox extension in browser</name>
  <files></files>
  <action>
Present the following verification steps to the user. The user must load the extension in Firefox and test all critical flows. Wait for user confirmation before proceeding.

**What was built:** Complete Firefox MV3 extension in dist/firefox/ -- background script with direct PDF pipeline calls, icon activation via tabs.onUpdated, IndexedDB graceful degradation.

**Steps to verify:**

Load the extension:
1. Open Firefox (version 128+)
2. Navigate to about:debugging#/runtime/this-firefox
3. Click "Load Temporary Add-on..."
4. Select dist/firefox/manifest.json
5. Confirm no errors in the browser console (F12 -> Console)

Test icon activation (FOX-03):
6. Navigate to https://patents.google.com/patent/US11234567B2/en
7. Verify the toolbar icon becomes colored (active)
8. Navigate to https://www.google.com
9. Verify the toolbar icon is gray (inactive)

Test citation generation (FOX-02):
10. Return to the patent page
11. Wait for the icon to become fully active (may take a few seconds for PDF fetch+parse)
12. Highlight some text in the patent specification (e.g., a sentence in the description)
13. Right-click and select "Get Citation"
14. Verify a citation popup appears with a column:line reference
15. Verify the citation format matches Chrome output (e.g., "col. 4, ll. 5-20")

Test with a different patent:
16. Navigate to https://patents.google.com/patent/US7654321B2/en
17. Repeat steps 11-15

Test IndexedDB degradation (FOX-05) -- OPTIONAL but recommended:
18. In Firefox Settings -> Privacy: set "Firefox will" to "Never remember history"
19. Restart Firefox
20. Load the temporary add-on again
21. Navigate to a patent page
22. Verify citation generation still works (may be slower without cache)
23. Check browser console -- should see "[FF] IndexedDB unavailable" warning but no errors
24. Reset Firefox history setting when done
  </action>
