<objective>
Close gap_inventory Bucket E (1 case: synthetic-gutter-1, REGEX_BUG).

The current `patentIdFromCaseId` function in tests/e2e/specs/regression.spec.js (lines 68-72) throws on the synthetic-gutter-1 case-id because the regex `^([A-Z]{2}\d+[A-Z]?\d*)-` requires a real patent prefix (US/EP + digits). The synthetic case is meant to validate gutter-number handling against a synthetic fixture (`tests/fixtures/synthetic-gutter.json`) — there is no real Google Patents page for it.

Two viable closure paths (pick ONE, document why):
1. **Skip path**: Mark synthetic cases as `test.skip()` in the regression spec with a comment pointing to a future dedicated synthetic-fixture spec. Pros: clean separation of concerns; the regression spec stays focused on live Google Patents replay. Cons: regression coverage of synthetic fixtures is deferred.
2. **Route path**: Branch in patentIdFromCaseId to return a sentinel value (`'__synthetic__'`); branch in the test body to skip `gotoPatent` and read citations from the fixture directly (no live page interaction). Pros: synthetic cases still execute in the regression suite. Cons: complicates the spec body; the synthetic path is mostly orthogonal to what the spec verifies (selection emulation on live HTML).

**Recommendation: take the SKIP path** — it's smaller, more honest about scope, and Phase 28+ will add a dedicated synthetic-fixture spec. The synthetic-gutter case is fundamentally not in scope for "live-page selection emulation."

Purpose: SEL-03 contract says "every patent matches baseline." With synthetic cases skipped (and the regex no longer throwing), the regression spec executes cleanly for 75/75 live cases (no early-abort).

Output: Updated regression.spec.js — either with `test.skip()` for synthetic cases (preferred) or with a sentinel-based fixture-only branch. Same target: regression spec no longer aborts on synthetic-gutter-1.
