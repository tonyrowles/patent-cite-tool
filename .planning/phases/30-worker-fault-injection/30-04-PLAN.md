Rewrite intercepted GET /cache URLs to use a unique `v=test-{nonce}` so the Worker returns 404 (cache miss), forcing the extension to fall through to /?patent= for USPTO.
3. **Verifier API mismatch:** `pdf-verifier.js` returns `status: 'pass'` or `'disagree'`; the spec asserted `'agree'` which never matches. Update the assertion.

Rewrite the helper so it takes a `BrowserContext` instead of a `Page`. Replace `page.route` with `context.route`. For each intercepted Worker request:

1. **Method GET on `/cache?` URL:** rewrite the `v=` query param to `v=test-{nonce}` (nonce is module-scope random string set once per test invocation). Then call `route.continue({ url: rewrittenUrl })`. This forces cache miss, the Worker returns 404, and the extension proceeds to its fallback (USPTO via /?patent=).
2. **Method POST on `/cache?` URL:** add `X-PCT-Test-Mode: true` to outbound headers; continue with `route.continue({ headers })`. Plan 30-01's Worker guard then suppresses the KV write.
3. **GET on `?patent=` URL (USPTO fetch):** continue unchanged. (Optionally also inject `X-PCT-Test-Mode: true` for safety; the Worker ignores it for this route.)
4. All non-matched requests: do NOT register handlers (let them proceed normally).

The helper still returns `{ getCallCount() }` exposing how many times it fired on the Worker URL (any method) for the canary assertion.

Exact signature:
```javascript
export function installWorkerTestModeRoute(context) {
  const nonce = `test-${Date.now()}-${Math.random().toString(36).slice(2, 8)}`;
  let callCount = 0;

  context.route('https://pct.tonyrowles.com/**', async route => {
    callCount += 1;
    const req = route.request();
    const url = new URL(req.url());
    const method = req.method();

    if (url.pathname === '/cache' && method === 'GET') {
      // Force cache miss by rewriting version
      url.searchParams.set('v', nonce);
      return route.continue({ url: url.toString() });
    }
    if (url.pathname === '/cache' && method === 'POST') {
      // KV write suppression (Plan 30-01 guard)
      const headers = { ...req.headers(), 'x-pct-test-mode': 'true' };
      return route.continue({ headers });
    }
    // Otherwise (?patent= USPTO fetch, OPTIONS preflight): proceed unchanged
    return route.continue();
  });

  return {
    getCallCount: () => callCount,
    nonce,
  };
}
```

Note: HTTP header names lowercased by Playwright/browsers in `route.request().headers()`. Use lowercase `x-pct-test-mode` for the `headers` object passed to `route.continue`. The Worker reads it with `request.headers.get('X-PCT-Test-Mode')` which is case-insensitive per HTTP spec.
