<objective>
Plans 30-02 and 30-04 confirmed that neither `page.route` nor `context.route` can intercept the Chrome extension's offscreen-document outbound requests. Add a minimal extension test hook — two `chrome.storage.local` keys read by offscreen.js — so the Playwright spec can force a cache miss (cache-bypass) and request KV-write suppression (test-mode) without needing CDP-level request interception.

Scope discipline rationale: This is functionally a TEST hook (same category as Phase 26's data-testid attributes). The keys are read but never written by production code paths; if absent, behavior is unchanged. The hook is ~10 lines of additive code in offscreen.js.
</objective>

<extension_hook_design>
**Storage keys (chrome.storage.local):**
- `pct_test_cache_version` (string, optional): When set, offscreen.js uses this string in the `v=` query param of GET /cache and POST /cache URLs (instead of the hardcoded `CACHE_VERSION = 'v3'`). Forces cache miss when the test sets a nonce. **Absent in production** → use `CACHE_VERSION`.
- `pct_test_mode` (boolean, optional): When `true`, offscreen.js adds `'X-PCT-Test-Mode': 'true'` to POST /cache request headers (so Plan 30-01's Worker guard suppresses the KV write). **Absent in production** → no header added.

**Read pattern (synchronous-ish):** Use a single `chrome.storage.local.get(['pct_test_cache_version', 'pct_test_mode'])` call inside each cache call site, awaited before constructing the URL/headers. Adds ~5ms; negligible.

**No-op proof:** Production extension never sets these keys (verified by `grep -r "pct_test_cache_version" src/` returning ONLY the read sites). If neither key is present, the storage.get resolves with `{}` and the code paths take the existing branches (`CACHE_VERSION` used; no header added).
</extension_hook_design>
</context>

```javascript
// tests/e2e/lib/worker-test-mode-route.js
//
// Phase 30 — Test-mode hook installer.
//
// CDP-level routing (page.route / context.route) cannot intercept the
// Chrome extension's offscreen-document outbound requests (confirmed in
// Plans 30-02 and 30-04). Instead, this helper sets two chrome.storage.local
// keys that offscreen.js reads at each /cache call site (added in Plan 30-05).
//
// Keys set:
//   - pct_test_cache_version: a per-test nonce that overrides v=CACHE_VERSION → forces cache miss
//   - pct_test_mode: true → offscreen.js adds X-PCT-Test-Mode: true on POST /cache
//
// Production extension never sets these keys; behavior identical to today when absent.

/**
 * Install the test-mode storage hook for a Playwright test.
 *
 * @param {import('@playwright/test').BrowserContext} context
 * @param {string} extensionId  - Chrome extension ID from extension-loader.js
 * @returns {Promise<{nonce: string, cleanup: () => Promise<void>}>}
 */
export async function installWorkerTestModeRoute(context, extensionId) {
  if (!context) throw new Error('installWorkerTestModeRoute: context is required');
  if (!extensionId) throw new Error('installWorkerTestModeRoute: extensionId is required');

  const nonce = `test-${Date.now()}-${Math.random().toString(36).slice(2, 8)}`;

  // Find the extension service worker
  const serviceWorkers = context.serviceWorkers();
  const extSw = serviceWorkers.find(sw => sw.url().includes(extensionId));
  if (!extSw) {
    throw new Error(`installWorkerTestModeRoute: service worker for extension ${extensionId} not found`);
  }

  // Set the storage keys via the service worker context
  await extSw.evaluate(async ({ nonce }) => {
    await chrome.storage.local.set({
      pct_test_cache_version: nonce,
      pct_test_mode: true,
    });
  }, { nonce });

  return {
    nonce,
    async cleanup() {
      await extSw.evaluate(async () => {
        await chrome.storage.local.remove(['pct_test_cache_version', 'pct_test_mode']);
      });
    },
  };
}
```
