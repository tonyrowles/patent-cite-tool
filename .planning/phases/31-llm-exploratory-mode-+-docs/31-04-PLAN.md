    ### Section 7: Troubleshooting
    Bulleted list of common failures and remedies:
    - **`claude` CLI not found** → `npm run e2e:explore` exits 3. Install Claude Code (https://docs.claude.com/claude-code) and run `claude login`.
    - **Subscription exhausted** → iteration classified as `LLM_API_ERROR`; `stderr` contains `subscription`, `quota`, or `credit`. Wait until next billing cycle or upgrade to a higher plan. The MAX_5_SUBSCRIPTION_EXHAUSTED signal is one of these stderr strings — exploratory mode is unavailable until the pool refills.

    - **Ledger cap reached** → `tests/e2e/.llm-spend-ledger.json` shows total_usd >= $100. Wait until next month (auto-rolls over) or delete the file.
    - **Chromium install failure** → `npx playwright install chromium` failed. See playwright.dev troubleshooting. Common cause: outdated Playwright cache; clear `~/.cache/ms-playwright/` and reinstall.
    - **USPTO rate limit** → fault-injection or exploratory iteration involving USPTO fallback gets HTTP 429. Slow down (`--iterations 1`) and retry after 60s.
    - **Cache hit shortcuts** — exploratory runs install a per-test nonce (`pct_test_cache_version`) that forces a cache miss for THIS request only. If you see suspicious instant citations (< 100ms), check `tests/e2e/.pdf-cache/` for stale entries; delete and retry.
    - **DOM drift on Google Patents** → spec fails with `GOOGLE_DOM_DRIFT`. Check the smoke spec output; if Google changed the page structure, regenerate selectors and update `tests/e2e/lib/navigation.js`.
    - **Hallucination rate spikes** — most iterations classify as `LLM_HALLUCINATED_SELECTION`. Possible causes: (a) the patent's spec text starts on a late page (try `extractSpecText(..., { maxPages: 20 })`); (b) pdfjs extraction differs from what the LLM was prompted with — file a Phase 31 follow-up issue.
    - **`llm-report.json` exists but is empty/half** — atomic write means it should always be valid JSON. If you see partial JSON, check disk space (`df -h`) and report a bug. Re-running the same `--iterations N` will append a new run under a new run-id.
