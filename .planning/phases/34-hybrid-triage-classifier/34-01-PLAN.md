<objective>
Add the `invokeClaudePWithLedger` wrapper to `tests/e2e/lib/llm-driver.js` as a Wave 1 foundation for the Phase 34 triage classifier. The wrapper is the SOLE allowed entry point for triage code into the `claude -p` subprocess: it composes the existing `invokeClaudeP` + `parseClaudeResponse` primitives with the existing `appendLedgerEntry` ledger writes, enforces the subscription-local invariant via an `process.env.CI`/`GITHUB_ACTIONS` gate, and short-circuits on monthly + per-phase spend cap blocks BEFORE spawning the subprocess. The ledger append fires unconditionally on every code path that reaches `invokeClaudeP` so that `is_error:true` responses with non-zero `total_cost_usd` are still recorded (Pitfall 8 mitigation from 34-RESEARCH.md). Vitest unit tests cover the three gated branches (CI, monthly cap, phase cap), the happy path, and the `is_error` ledger-still-fires invariant.

Purpose: Plans 02, 03, 04, 05 all depend on this wrapper existing. Plan 03 imports it as the default `invokeLlm` injected dep of `runTriage`; Plan 04 imports it into the CLI runner; Plan 05 forbids the bypass path at edit time via ESLint. The wrapper itself ships zero NEW logic — every primitive is composed from Phase 31/32 modules per the 34-RESEARCH.md "Don't Hand-Roll" table.

Output: One modified module file with a new named export, one extended unit test file with a new describe block, both green under `npm run test:src`.
