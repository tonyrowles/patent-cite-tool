## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LLM-derived text → issue body | `triageFinding.rationale` and `iteration.verifier_verdict.reason` originate from LLM output (untrusted) and render in GitHub Markdown |
| Builder inputs → output string | All inputs are JSON-derived; output is a single string passed to `gh issue create --body-file -` |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-35-01-01 | Tampering | `triageFinding.rationale` markdown injection (e.g., `[click](http://evil)` or `## HEADER`) | mitigate | Wrap LLM-derived rationale in fenced code block via `fenceCode(...)` helper (T-29-02-2 pattern); Test 10 pins the defense |
| T-35-01-02 | Tampering | `iteration.verifier_verdict.reason` markdown injection | mitigate | Same fenced-code-block wrap inside the Verifier Disagreement section |
| T-35-01-03 | Denial of Service | Adversarial 10K-char LLM rationale blowing past 65,536-char GitHub body limit | mitigate | Per-section budgets (D-03) enforced inside the builder; Test 5/6/7/8 pins each budget; worst-case body ≤ 50K (Pitfall 2) |
| T-35-01-04 | Information Disclosure | Fingerprint truncated past 65,536-char limit causing dedup break | mitigate | D-02 line-1 placement; Test 2/3 pin `body.split('\n')[0]` and `body.indexOf('<!-- fp: ') === 0` (Pitfall 1) |
| T-35-01-05 | Repudiation | Non-deterministic builder output (different test runs → different bodies → different fingerprints → false dedup misses) | mitigate | D-04 PURE constraint; Test 1 pins determinism via two-call comparison |
