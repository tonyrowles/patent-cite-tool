| Boundary | Description |
|----------|-------------|
| GitHub issues → digest | Untrusted-ish issue titles/labels flow from the GitHub API into shell (`gh`) and into the committed markdown file |
| digest → gh shellout | Dynamic title/category/body cross into `gh issue create` / `gh api graphql` |
| CI env → ledger read | Ledger is absent in CI; absence must degrade gracefully, never bypass a cap |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-37-02-01 | Tampering / Elevation | `gh issue create` title interpolation | mitigate | Title shell-escaped `.replaceAll('"','\\"')`; body via `--body-file -` stdin (never interpolated) — e2e-report-issue.mjs:495 idiom |
| T-37-02-02 | Tampering | digest body reflects issue titles into a committed file | mitigate | Body is machine-generated from a closed-taxonomy label set + numeric counts; only escaped issue-derived strings reach the file; no free-form user text |
| T-37-02-03 | Information disclosure / DoS | ledger absent in CI rendered as $0 (masking) | mitigate | `fs.existsSync(LEDGER_PATH)` FIRST → 'cost data unavailable' (D-15, Pitfall 2); never set E2E_LEDGER_PATH_OVERRIDE (throws in CI, llm-ledger.js:85) |
| T-37-02-04 | Tampering | GraphQL query string injection (dormant path) | mitigate | All dynamic values passed via `-F`/`-f` variable bindings ($r/$c/$t/$b), never concatenated into the query (RESEARCH Code Examples) |
| T-37-02-05 | Tampering | category label string into shell | mitigate | Category derives from labels[0].name (Phase 34/35 closed ERROR_CLASSES taxonomy); used as data in the tally, not interpolated into a privileged command |
