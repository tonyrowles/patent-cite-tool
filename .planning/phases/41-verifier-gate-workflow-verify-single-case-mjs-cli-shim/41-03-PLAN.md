<objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub event (PR open/synchronize) -> workflow | The PR head SHA, head branch, body, and number are all PR-author-controlled. The workflow MUST treat them as untrusted input (env-hop pattern per Phase 40-03 CR-02). |
| PR branch checkout -> verifier execution | PR-side files include the modified verifier (potentially tampered). Layer 4 defense (verifier-pin to origin/main) makes PR-side mutations invisible to the gate. |
| PR diff -> diff-guard regex bank | PR-author-controlled file paths in the diff are validated against the canonical FORBIDDEN_PATHS bank (Plan 41-01 helper). |
| gh CLI -> GitHub API | Uses secrets.GITHUB_TOKEN (runner-injected, workflow-scoped, expires per run). No PATs (Pitfall 4 invariant). |
