<objective>
Lift Phase 42's CI-validated `scripts/auto-fix.mjs` dispatcher into a
GitHub Actions workflow triggered by `issues.labeled('triage')`. The
workflow opens a draft PR via `peter-evans/create-pull-request@v8` with
a PR body containing the load-bearing `<!-- affected_cases: id1,id2 -->`
HTML comment that Phase 41's verifier-gate consumes. The single
non-obvious architectural decision is the **two-commit split**:
the ledger update is committed to `main` directly (with `[skip ci]`)
BEFORE `cpr@v8` snapshots the working tree, so the auto-fix PR diff
contains only the source-code fix — clean against Phase 41's
diff-guard regex bank (which rejects PRs touching the ledger path).

dispatcher locally; Phase 43 makes it CI-driven so a real
`triage`-labeled issue produces a draft PR end-to-end. Phase 47's
HUMAN-UAT (a) (deferred from Phase 42) will be the live demo.

From `scripts/auto-fix.mjs` (Phase 42 — DO NOT MODIFY in this plan):
- CLI: `node scripts/auto-fix.mjs --issue <n> [--force-api] [--no-push] [--dry-run]`
- Exit codes:
    0 = success (applied OR dry-run OR skip-class short-circuit OR ls-remote idempotency hit)
    1 = rejected (parseFencedDiff / diff-guard / git apply --check / git push failed)
    2 = arg/contract error (missing --issue, no fingerprint, no/multi ERROR_CLASS)
    3 = fix_attempts cap reached (dispatcher already added human-review-required label + posted comment)
- Side effects on exit 0 with --no-push:
    - leaves HEAD on the auto-fix branch (so `git rev-parse --abbrev-ref HEAD` returns the branch)
    - writes a ledger entry to `tests/e2e/.llm-spend-ledger.json` (so the working tree has the ledger dirty)
    - prints to stdout: `[auto-fix] branch staged locally; push manually with: git push -u origin auto-fix/<n>-<fp8>`
    - prints to stdout: `[auto-fix] suggested PR-create command:\n  gh pr create --draft ... --body '<!-- affected_cases: ... -->\nfingerprint: ...\nfix_attempts: ...\nmodel: ...\ntransport: ...'`
- `RECOGNIZED_LABELS` set: `[...ERROR_CLASSES, 'PASS']` where ERROR_CLASSES is the closed RPT-02 taxonomy:
