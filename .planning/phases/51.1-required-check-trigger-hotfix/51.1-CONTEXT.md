<domain>
## Phase Boundary

Restore PR throughput on `tonyrowles/patent-cite-tool` by making `verifier-gate` and `deps-update-gate` (both made required by Phase 50 GATE-01) actually fire on PRs into `main`. Currently neither workflow triggers on `main`-targeted PRs, so the required checks never report, so every PR is blocked indefinitely.

**Fix approach: α — fast-path SUCCESS for non-matching PRs** (operator-decided):
1. **`.github/workflows/v40-verifier-gate.yml`** — Remove the `branches: ['auto-fix/*']` BASE-ref filter from the `on: pull_request:` block so the workflow fires on every PR into `main`. Add a "scope decision" step at the top of the `verifier-gate` job (and any other job that should fast-path) that checks `github.head_ref`; if it doesn't start with `auto-fix/`, the remaining steps are skipped and the job exits 0 (SUCCESS check_run posts). Heavy work runs only for `auto-fix/*` PRs as designed.
2. **`.github/workflows/v40-deps-update.yml`** — Add a `pull_request:` trigger (currently only `schedule:` + `workflow_dispatch:`). Apply the same scope-decision fast-path inside the `deps-update-gate` job (fast-path SUCCESS for non-`v40-deps-update/*` head refs).
3. Land the fix via break-glass (Phase 50 §7): temporarily re-add operator bypass, push fixes directly to main (no PR needed — bypass actor can push), open ONE verification test PR to confirm both checks now fire, CLOSE the test PR, remove the bypass actor to restore Phase 50 SC-2.

**Out of scope:**
- Adding additional jobs to the workflows (e.g., a separate "skip-reporter" workflow). Decided against per design Option γ rejection.
- Removing verifier-gate / deps-update-gate from required_status_checks (Option β rejection — preserves Phase 50 trust enforcement).
- Touching the `diff-guard` job filter — it should already fire on all PRs after the BASE-ref filter is removed; that's an intentional side-benefit (LOCKED-path protection becomes universal, which is what UAT-47-e was trying to verify).
- Re-running UAT-47-e and UAT-47-a after the fix lands — that's a v4.2 Phase 56 follow-up per the existing STATE pending-todo.

<decisions>
## Implementation Decisions

### Fix mechanism

- **D-01:** Remove `branches: ['auto-fix/*']` from `.github/workflows/v40-verifier-gate.yml` lines 47 (`pull_request:`)+. Workflow then fires on every PR into the default branch.
- **D-02:** Add `pull_request: { types: [opened, synchronize, reopened] }` to `.github/workflows/v40-deps-update.yml` `on:` block (sibling to existing `schedule:` + `workflow_dispatch:`).
- **D-03:** Add a "scope decision" GitHub Actions step at the very top of EACH of these jobs in v40-verifier-gate.yml: `verifier-gate`, `regression-suite`, `ready-flip` (not `diff-guard` — that's the universal LOCKED-path check). Step pattern:
  ```yaml
  - name: Scope decision (auto-fix/* PRs only; fast-path SUCCESS otherwise)
    id: scope
    if: github.event_name == 'pull_request'
    run: |
      if [[ "${{ github.head_ref }}" == auto-fix/* ]]; then
        echo "active=true" >> "$GITHUB_OUTPUT"
      else
        echo "active=false" >> "$GITHUB_OUTPUT"
        echo "Fast-path SUCCESS — non-auto-fix PR (head=${{ github.head_ref }})"
      fi
  ```
  All subsequent steps within each job add `if: steps.scope.outputs.active == 'true' || github.event_name != 'pull_request'`. (The second clause preserves backward-compat for any non-pull_request triggers the workflow might have — even though it currently only has `pull_request`.)
- **D-04:** For `.github/workflows/v40-deps-update.yml` `deps-update-gate` job — same scope-decision pattern with `${{ github.head_ref }}` check against `v40-deps-update/*` prefix. dep-scan job retains its current scope (schedule + workflow_dispatch only) since it's the PR-creator, not a PR-gate.

### Break-glass + Push strategy

- **D-05:** Use Phase 50 §7 break-glass runbook to temporarily re-add the operator bypass actor (id 254599900, `bypass_mode=always`). This re-allows direct pushes to main during the fix window.
- **D-06:** Push the YAML fix commits DIRECTLY to main while bypass is active (no PR — would Catch-22 itself since the fix is what unjams PRs). Single combined commit covering both YAML edits + any Vitest test updates.
- **D-07:** Re-test verifier-gate trigger by opening ONE verification test PR on a throwaway branch (`test-only/trigger-hotfix-verify-${ts}`) touching `.planning/scratch/trigger-hotfix-probe.txt` (non-CODEOWNED, non-LOCKED). Assert: BOTH `verifier-gate` AND `deps-update-gate` contexts appear in `gh pr checks` output. Whether they pass or fail is secondary; what matters is they FIRE. Close PR + delete branch immediately after evidence capture (UAT-47-e / GATE-03 convention).
- **D-08:** AFTER verification test PR confirms both checks fire, remove the bypass actor again to restore Phase 50 SC-2 (`bypass_actors=[]` end-state). Capture pre+post ruleset state to evidence/.
