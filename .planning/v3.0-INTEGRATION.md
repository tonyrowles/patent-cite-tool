## E2E Flow Verdict

**Deterministic flow (local-dev + nightly CI):**

```
Developer runs `npm run e2e:regression`                    (package.json L10)
   ↓ npm run build:chrome
   ↓ playwright test specs/regression.spec.js
   ↓ regression.spec.js iterates 76 golden cases
     - loadExtension / setTriggerMode / gotoPatent / selectText (Phase 26/27)
     - getCitation (Phase 26)
     - verifyCitation (Phase 28)
     - appendCase(report, { ..., verifier_verdict }) (Phase 27/28)
   ↓ report.json at tests/e2e/artifacts/${RUN_ID}/report.json
```

**Same flow on CI nightly cron (`.github/workflows/e2e-nightly.yml`):**

```
06:00 UTC schedule
   ↓ smoke pre-flight (1 case) — meta-issue on DOM drift
   ↓ select-cron-cases.mjs picks 30-case rotation (or --full for Sunday/dispatch)
   ↓ regression.spec.js + fault-injection.spec.js
   ↓ scripts/e2e-report-issue.mjs reads report.json
   ↓ files per-case issues via fingerprint dedup (Phase 29)
```

**Exploratory LLM flow (local-only, hard $100 cap, CI-guarded):**

```
Developer runs `npm run e2e:explore -- --iterations N`     (package.json L13)
   ↓ scripts/e2e-explore.mjs
   ↓ CI guard (exits 1 if CI || GITHUB_ACTIONS set) — Phase 31 LLM-07
   ↓ checkSpendCap (hard cap = $100/month) — Phase 31 LLM-06
   ↓ for each iteration: runOneIteration uses 8 cross-phase primitives
     (boundary 5 above) → writes llm-report.json (separate file)
```

**Naming caveat (documentation only, not a wiring gap):**

The integration prompt referenced `npm run e2e:deterministic`. The actual
package.json script names are `e2e:smoke` / `e2e:regression` / `e2e:silent`.
`tests/e2e/README.md` L17 groups all three under the heading "Deterministic"
mode. Consumers (CI workflow L131-L139, README L80) reference the correct
existing names. No code change required; the prompt's `e2e:deterministic` is
a category label, not a script name.

**Count caveat:** report.json carries 76 entries per run (3 skipped synthetic +
2 skipped deferred + 66-71 live, depending on rotation). The "76-case golden
suite" and "66 live cases" figures are both accurate at different layers
(report contract vs. live execution).

**E2E flow verdict: COMPLETE — workflow ends-to-end as designed for all three
entry points (local deterministic, CI nightly deterministic, local LLM
exploratory).**
